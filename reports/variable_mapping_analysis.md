## 3. 구조적 불일치 분석

### Type A: 실험에는 입력인데 웹앱에 해당 입력 없음

| 실험 변수 | 영향도 | 설명 |
|-----------|--------|------|
| **해당 없음** | - | 실험의 모든 입력 변수가 웹앱에도 존재함 (단위 차이만 있음) |

> 결론: 실험의 가동 조건을 웹앱에서 완전히 재현할 수 있음

### Type B: 웹앱에는 입력인데 실험에서 기록/변동하지 않는 항목

| 웹앱 입력 | 영향도 | 설명 |
|-----------|--------|------|
| **온도 (Temperature, °C)** | 경미 | 실험은 상온(~25°C) 고정이지만 별도 기록 없음. 웹앱 기본값 25°C로 설정하면 문제 없음 |
| **pH 모드 (목표 pH / 고정 NaOH)** | **중요** | 웹앱 고유 개념. 실험에서는 NaOH를 투입하고 pH는 결과로 나옴 → "고정 NaOH" 모드가 실험과 구조적으로 일치. "목표 pH" 모드는 역산 기능이므로 실험과 역할이 다름 |
| **목표 pH (Target pH)** | **중요** | "목표 pH" 모드에서만 사용되는 입력. 실험에서 pH는 출력이므로 이 입력값에 대응하는 실험 변수 없음. (→ Type C 역할 역전 참조) |
| **NaOH 전략 (uniform/front_loaded/custom)** | 경미 | 실험은 사포니피케이션 방식이므로 NaOH를 Fresh Org에 한 번 투입. 웹앱에서 사포니피케이션 모드를 선택하면 이 전략은 무관 |
| **파라미터 프로필 선택** | 웹앱 전용 | 모델 튜닝 기능으로 실험과 무관 |
| **파라미터 수동 편집 (pH50, k 등)** | 웹앱 전용 | 모델 튜닝 기능으로 실험과 무관 |
| **황산염 농도 (C_sulfate)** | 경미 | 웹앱에서 자동 계산, 실험에서 측정 안 함. 내부 보정용이므로 사용자 입력 불필요 |
| **경쟁추출/종분화 토글** | 웹앱 전용 | 모델 옵션으로 실험과 무관 |

### Type C: 실험에서는 출력인데 웹앱에서는 입력 (역할 역전) ⚠️

| 변수 | 실험에서의 역할 | 웹앱에서의 역할 | 영향도 | 설명 |
|------|---------------|---------------|--------|------|
| **각 단의 pH** | **출력** (NaOH 투입 후 자연적으로 결정되는 측정값) | **입력** ("목표 pH 모드"에서 target_pH로 사용) | **치명적** | 이것이 가장 핵심적인 구조적 불일치. 실험에서 pH는 NaOH 투입량의 *결과*인데, 웹앱의 "목표 pH" 모드에서는 원하는 pH를 *지정*하고 필요한 NaOH를 역산함. 검증 스크립트(`field_validation_main.py`)에서도 실측 pH를 target_pH_per_stage로 넣어 시뮬레이션하는데, 이는 실험 출력을 시뮬레이션 입력으로 쓰는 역전 구조 |

> **권고**: 실험 데이터와의 정확한 비교를 위해서는 반드시 **"고정 NaOH" 모드**를 사용해야 함.
> 이 모드에서는 NaOH 유량과 농도를 입력으로 넣고, pH와 금속 농도가 모두 출력으로 나와 실험 구조와 동일해짐.

### Type D: 실험에서 측정하지만 웹앱에서 출력하지 않는 항목

| 실험 측정값 | 영향도 | 설명 |
|------------|--------|------|
| **Cr 농도** | 무시 가능 | 모든 실험에서 Cr < 1 mg/L. 웹앱은 7종 금속(Li,Ni,Co,Mn,Ca,Mg,Zn)만 다루며 Cr 미포함. 실질적 영향 없음 |
| **2단 Raffinate (Ni_Cy)** | 해당 없음 | 실험에서도 미측정. 웹앱은 모든 단의 데이터를 TAB 5에서 출력하므로 실험에서 측정하면 비교 가능 |

> 결론: 실험에서 측정하는 모든 유의미한 변수를 웹앱에서도 출력 가능

### Type E: 단위/형식 차이

| 변수 | 실험 단위 | 웹앱 단위 | 변환 계수 |
|------|----------|----------|----------|
| 금속 농도 | mg/L | g/L | ÷ 1000 |
| 유량 | mL/min | L/hr | × 0.06 |
| NaOH 농도 | wt% | wt% 또는 M | wt% 선택 시 동일, M 선택 시 변환 필요 |

> 주의: 웹앱에서 실험값을 직접 입력할 때 사용자가 수동으로 단위를 변환해야 함. 웹앱 UI에 mL/min, mg/L 단위 입력 옵션이 없음.

### Type F: 웹앱에서 출력하지만 실험에서 측정하지 않는 항목

| 웹앱 출력 | 실험 측정 여부 | 비교 가능성 |
|-----------|--------------|------------|
| 금속별 추출률 (%) | 간접 계산 가능 (Feed-Raff 차이로) | 가능 |
| NaOH 소비량 (mol/hr, stage별) | 미측정 (투입량만 기록) | **불가능** — 웹앱은 실제 소비량을 계산하지만, 실험에서는 투입량만 알 수 있음 |
| 로딩률 (%) | 미측정 | **불가능** |
| 유기계 금속 농도 (C_org_out) | 미측정 (CSV에 없음) | **불가능** — Loaded Org 분석 데이터가 없음 |
| 물질수지 검증 | 수동 계산 필요 | 간접 가능 |
| McCabe-Thiele 다이어그램 | 해당 없음 | 비교 대상 없음 |
| pH Isotherm 곡선 | 피팅 데이터 필요 | 피팅 탭(TAB 6)에서 실험 데이터 업로드 시 비교 가능 |
| 피팅 파라미터 (pH50, k, E_max) | 직접 측정 불가 | pH vs E% 데이터를 피팅 탭에 업로드해야 도출 가능 |

## 4. 종합 매칭도 평가

### 4.1 매칭 현황 요약

```
실험 입력 변수 10개 → 웹앱 입력으로 100% 매핑 가능 (단위 변환 필요)
실험 출력 변수 4개  → 웹앱 출력에서 3개 직접 비교 가능, 1개 미포함(Cr)
웹앱 입력 변수 17개 → 실험 대응 10개, 웹앱 전용 7개
웹앱 출력 변수 30개 → 실험에서 검증 가능 ~10개, 나머지는 실험 미측정
```

### 4.2 핵심 불일치 3건

| 순위 | 불일치 | 유형 | 영향도 | 개선 방안 |
|------|--------|------|--------|----------|
| **1** | pH 역할 역전 (실험=출력, 웹앱=입력) | Type C | **치명적** | "고정 NaOH" 모드 사용 필수. 또는 웹앱에 "NaOH 투입 모드에서 pH 예측" 결과를 실험 pH와 비교하는 전용 검증 기능 추가 |
| **2** | 유기계 농도 미측정 | Type F | 중요 | 실험에서 Loaded Org 분석 추가하면 물질수지 완전 검증 가능. 현재는 수계 데이터만으로 부분 검증 |
| **3** | 단위 차이 (mg/L↔g/L, mL/min↔L/hr) | Type E | 경미 | 웹앱 UI에 단위 선택 옵션 추가 또는 변환 안내 표시 |

### 4.3 구조적으로 양호한 항목

- 금속 종류: 실험 8종 중 실질적 7종(Cr 제외)이 웹앱과 동일
- 추출제/농도/단수: 완전 일치
- NaOH 방식 (Saponification): 양쪽 모두 지원
- Stage별 Raffinate 농도: 웹앱 TAB 5에서 비교 가능
- Stage별 pH: 웹앱 TAB 1 pH 프로파일 + TAB 5에서 비교 가능

## 5. 검증 시 권장 모드

### 실험 데이터와 직접 비교를 위한 웹앱 설정

```
pH 모드        : "고정 NaOH" (실험과 동일한 입출력 구조)
NaOH 모드      : "사포니피케이션"
NaOH 농도      : 실험값 그대로 (wt% 선택)
NaOH 유량      : 실험값 (mL/min × 0.06 = L/hr)
Feed 금속 농도  : 실험값 (mg/L ÷ 1000 = g/L)
Feed 유량       : 실험값 (mL/min × 0.06 = L/hr)
추출제/농도/단수 : 실험값 그대로
온도            : 25°C (실험 상온 가정)
```

### 비교 가능한 출력 항목
