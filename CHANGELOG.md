## [v2.1.1] - 2026-03-07

### 🧭 검증 투명성 및 해석 가이드 강화

- **모델 보정 프로필 선택 기능 추가**:
  - 사이드바에서 `현장 보정 (Isd 108)`과 `문헌 기본값` 프로필을 전환할 수 있도록 개선했습니다.
  - 사용자는 현장 적합형 결과와 범용 비교용 기준선을 구분해서 해석할 수 있습니다.
- **검증 커버리지 안내 추가**:
  - 현재 입력 조건이 Data1~6 현장 검증 범위와 얼마나 겹치는지 사이드바와 결과 탭에서 직접 표시합니다.
  - pH, 추출제 농도, 온도, 단수, 고로딩 상태를 기준으로 해석 주의사항을 안내합니다.
- **수렴 상태 표시 강화**:
  - 결과 탭과 사이드바에 다단 계산 `수렴/미수렴` 상태와 반복 횟수를 표시하도록 개선했습니다.
  - 사용자가 계산 결과를 해석하기 전에 solver 상태를 먼저 확인할 수 있습니다.

### 📊 검증/비교 정합성 개선

- **검증 스크립트의 황산염 계산식을 대시보드와 정렬**:

    입력 금속 조성으로부터 대시보드와 동일한 방식으로 황산염 농도를 계산합니다.
- **데이터셋별 오차 요약 추가**:

- **추출제 비교 탭 비교 조건 명시**:
  - 비교 탭 상단에 공통 `C_ext`, 단수, 온도, 황산염 농도를 표시해,
    어떤 조건에서 두 추출제를 비교하는지 즉시 확인할 수 있게 했습니다.

## [v2.1.0] - 2026-03-07

### 🔧 시뮬레이션 엔진 정밀화 및 코드 품질 개선

- **🔄 고정 NaOH 모드 수렴 대폭 개선**:
  - `_solve_with_fixed_NaOH`에 로딩 감쇠 내부 반복(최대 15회)을 추가하여, 기존 bisection 반복마다 이전 pH의 로딩률을 사용하던 부정확성을 해결했습니다.
  - 다단 역류 solver의 relaxation factor를 4단계 적응형(0.2→0.5→0.7→0.9)으로 개선했습니다.
  - **개선 결과**: 기존 500회 반복 미수렴 → **62회 수렴** 달성.
- **⚛️ n_eff 가중 평균 계산 정밀화**:
  - 고로딩 다핵 착물 알고리즘의 최대 이론 적재량 계산에서 `C_ext/2.0` 하드코딩(모든 금속 n_ext=2 가정)을 제거하고, 실제 유기상 금속 구성 기반 가중 평균 n_ext를 사용하도록 개선했습니다.
  - Li(n_ext=1)가 지배적인 로딩 시나리오에서의 정확도가 향상됩니다.

### 🧹 코드 품질 개선

- **`ao_ratio` 미사용 파라미터 제거**: `distribution_coefficient()` 함수에서 사용되지 않던 `ao_ratio` 파라미터를 제거하고, pseudo-D의 의미를 docstring에 명확히 기술했습니다.
- **도달 불가 코드(dead code) 삭제**: `multistage_sx.py`에서 `use_bisection`이 항상 False로 고정되어 실행되지 않던 bisection 분기(약 30줄)를 제거했습니다.
- **미등록 금속/추출제 방어**: `extraction_efficiency()` 함수에 입력 검증을 추가하여, 등록되지 않은 금속이나 추출제 사용 시 `KeyError` 대신 명확한 `ValueError` 메시지를 반환하도록 개선했습니다.
- **목표 pH 모드 구조적 한계 문서화**: NaOH 역산 구조에서 Q_aq_eff 희석 미반영 및 pH>7 OH⁻ 보정 비대칭에 대한 주의사항을 docstring에 명시했습니다.

### 📊 UI 개선

- **추출제 비교 탭 조건 불일치 수정**: `C_sulfate` 파라미터 누락을 수정하여 메인 시뮬레이션과 동일한 조건으로 비교하도록 했습니다.
- **추출제 비교 탭 C_ext 개선**: 하드코딩(0.5M/0.64M)을 제거하고 사이드바에서 설정한 추출제 농도를 사용하도록 변경했습니다.
- **피팅 탭 변수명 충돌 방지**: `result` → `fit_result`로 변경하여 메인 시뮬레이션 결과 덮어쓰기 위험을 제거했습니다.
- **분리계수 표시 개선**: D값이 양쪽 모두 0인 경우 `inf` 대신 `N/A`로, 한쪽만 0인 경우 `> 10⁶`로 표시하도록 개선했습니다.

### 📋 검증 결과

## [v2.0.0] - 2026-03-06

### ✨ 고농도(High-Loading) 특화 메커니즘 고도화 및 Phase 3 기본화

- ⚛️ **고로딩 환경 원수(High-Loading) 다핵 착물(Polynuclear Species) 방어 알고리즘 구현**:
  - 기존 Phase 3 경쟁 추출 수식의 한계로 인해, Data5(Ni 71g/L 등)와 같이 추출제가 극도로 부족한 고농도 조건에서 니켈이 전혀 추출되지 않던("과억제") 치명적 오류를 혁신적으로 수정했습니다.
  - **문헌 근거**: D2EHPA와 Cyanex 272는 추출제가 부족한 환경에서 금속 이온 여러 개를 공유하는 "다핵 올리고머(Multi-nuclear oligomeric complexes)"를 형성하여 원소 당 필요한 추출제의 배위수(stoichiometry)가 감소하는 특성을 보입니다.
  - **로직 구현**: 시뮬레이터 수식 내에서 유기상의 금속 점유율(Loading Ratio)이 포화될수록, 유효 화학양론 결합 계수($n_{\text{eff}}$)가 원래 값(2)에서 최대 60%(0.8 수준)까지 동적으로 비선형(cubic 함수) 감쇠하도록 알고리즘을 개편했습니다.
  - **검증 성과**: 극단적 고로딩 현장 실험 Data5 조건에서 Ni 후액(Raffinate) 농도 오차를 약 **39g/L ➡ 0.1g/L** 수준으로 완벽하게 수렴시키는 데 성공했습니다.
    - **[Data5 현장 실험 대비 개선 결과 증빙]**
      - Ni 투입 원수 농도: 71.0 g/L (상과상 희석)
      - **Ni 후액 (Raffinate)**: 현장 실험치 **16.0 g/L** ➡ 시뮬레이터(v2.0) **15.88 g/L** (-0.12g 오차로 완벽 정합)
      - Co 후액: 현장 0.16 g/L ➡ 시뮬레이터 0.000 g/L (허용 오차 내)
      - Mg 후액: 현장 0.00 g/L ➡ 시뮬레이터 0.001 g/L (허용 오차 내)
- ⚙️ **Phase 3 필수 기능화 (플래그 통폐합 및 UI 개선)**:
  - 수계 금속 간 "추출제 풀 공유 경쟁 모델"과 "수계 수산화물/황산염 종분화 완충 효과"를 사이드바 선택 옵션에서 제거하고, 시뮬레이션 핵심 엔진에 **디폴트(기본 활성)로 통합**하여 전반적인 시뮬레이션 정합성을 극대화했습니다.
- 📖 `sx_dashboard.py` 내 **📚 파라미터 및 문헌 (Tab 10)** 항목에 다핵 착물 형성에 따른 $n_{\text{eff}}$ 감쇠 매커니즘 문헌 근거 및 논문 제목 추가.
- 📄 자세한 결과 보고서 및 알고리즘 기획 내용은 `docs/verification/v2.0.0_고로딩_다핵착물_알고리즘_기획및결과보고서.md` 에 편철 완료.

### ✨ 고도화 및 신규 기능 (Proton Balance)

- ⚖️ **NaOH 총량 분배 전략 및 목표 pH 최적화**
  - 고정 NaOH 모드 시 균등(Uniform), 전단집중(Front-loaded), 커스텀 비율 분배 지원
  - 단일 목표 pH 설정 시, 이분법(Bisection)을 사용하여 다단 시뮬레이션의 최종(Raffinate) pH가 목표치에 도달하도록 총 NaOH 자동 최적화
- 🧪 **Proton Balance 물질수지 정밀화**
  - 수계 버퍼 용량(Buffer Capacity) 모델 추가: 황산염(Sulfate) 해리 평형(pKa=1.99)에 기반하여 pH 변화 저항력 반영 (사이드바 황산염 농도 설정 추가)
  - 추출제 사포닌화(Saponification)에 의한 염기(OH⁻) 소모량을 물질수지에 반영하여 고pH 영역 계산 불일치 해결
  - 다단 고정 NaOH 모드에서 발생하는 pH 진동 발산을 개선하기 위한 Bisection Solver 도입
- 📉 **McCabe-Thiele 다이어그램 기능 추가**
  - 특정 금속에 대하여 Isotherm 평형 곡선(Equilibrium Curve)과 조작선(Operating Line), 역류 Stage 단수 프로파일을 직관적으로 시각화하는 탭 추가
- 📊 **차트 시각화 강화**
  - 단수별 pH 프로파일 및 NaOH 시각화에서 'Feed' 지점 추가. pH 증감을 한눈에 파악 가능

### 💡 1.6.0 업데이트 도입 효과

- **실제 조업과 유사한 완만한 pH 프로파일 모사**: 황산염 버퍼(Buffer Capacity)가 반영되어 급격한 pH Drop(곤두박질치는 현상) 방지
- **고(高) pH 영역 계산 정확도 비약적 향상**: 사포닌화 시 소모되는 염기(OH⁻) 양을 물질수지에 정밀하게 반영
- **목표 pH 도달 연산 최적화 및 안정성 확보**: Bisection Solver 도입으로 발산(Oscillation) 현상을 완벽히 해결하고, 빠르고 정확한 최적 총 NaOH 투입량 도출 가능
