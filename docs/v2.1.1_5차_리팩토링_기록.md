# v2.1.1 5차 리팩토링 기록

작성일: 2026-03-08

## 목적

5차 리팩토링의 목적은 `sx_dashboard.py` 내부에 길게 이어져 있던 동적 탭 렌더링 코드를 별도 모듈로 분리하는 것이다.

이번 단계에서 해결하려던 문제:

- 탭별 UI 로직이 하나의 파일에 길게 붙어 있어 읽기 어려움
- 특정 탭 수정이 대시보드 전체 diff와 충돌을 유발함
- 탭 간 숨은 지역 변수 의존성이 유지보수를 어렵게 만듦

## 변경 파일

- `sx_simulator/dashboard_tabs.py` 신규 추가
- `sx_dashboard.py` 수정
- `PROJECT_STRUCTURE.md` 수정
- `docs/v2.1.1_5차_리팩토링_기록.md` 신규 추가

## 주요 변경 내용

### 1. 탭 렌더러 모듈 분리

새 파일 `sx_simulator/dashboard_tabs.py`를 추가하고 다음 탭 렌더링을 옮겼다.

- `render_results_tab()`
- `render_isotherm_tab()`
- `render_mccabe_thiele_tab()`
- `render_compare_tab()`
- `render_detail_tab()`

이 함수들은 기존 `sx_dashboard.py` 안에 있던 UI 코드를 거의 그대로 옮기되, 필요한 입력을 인자로 명시적으로 받도록 정리했다.

### 2. 메인 대시보드 파일 단순화

`sx_dashboard.py`는 이제 위 탭들에서 직접 긴 UI 블록을 렌더하지 않고, 각 탭에서 렌더러 함수를 호출한다.

즉, 메인 파일의 역할은 더 명확해졌다.

- 세션 상태 초기화
- 입력 수집
- 시뮬레이션 실행
- 탭 함수 호출
- 남은 일부 고급 탭 렌더링

### 3. 숨은 탭 간 의존성 제거

기존에는 앞선 탭에서 만들어진 `color_list`, `pH_range` 같은 값이 뒤쪽 탭에서 간접적으로 재사용되는 부분이 있었다.

이번 리팩토링에서:

- 탭 렌더러는 필요한 값(`metal_colors`, `ph_plot_range`)을 명시적으로 받도록 변경
- 수식 탭은 자체적으로 색상 목록을 만들도록 수정

이렇게 해서 탭 실행 순서나 지역 변수 생존 여부에 의존하는 구조를 줄였다.

## 동작 보존 원칙

이번 단계는 UI 코드의 위치와 구조만 바꾼 것이다.

바꾸지 않은 항목:

- 추출률 계산식
- 분배계수 계산
- 다단 시뮬레이션 로직
- 현장 검증 로직
- 사용자가 보는 입력 항목과 결과 내용

즉, 계산 결과와 시뮬레이션 능력은 그대로 유지하는 것을 전제로 했다.
