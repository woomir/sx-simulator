# v2.1.1 연구원 논의자료

## 문서 목적

이 문서는 현재 SX 시뮬레이터의 상태를 연구원과 빠르게 공유하고, 다음 개선 단계에서 어떤 부분이 코드로 해결 가능한 영역인지, 어떤 부분은 도메인 판단과 현장 정보가 필요한지 정리하기 위한 회의용 자료다.

핵심 질문은 세 가지다.

1. 이 시뮬레이터가 현재 잘하는 것은 무엇인가
2. 어디까지를 현재 신뢰 가능한 적용 범위로 볼 것인가
3. 남은 한계를 줄이기 위해 무엇이 더 필요한가

---

## 0. 먼저 이해해야 할 쉬운 표현

### 0.1 MAE란?

`MAE`는 `Mean Absolute Error`의 약자다.

쉽게 말하면:

- 시뮬레이션 결과와 실제 측정값이 평균적으로 얼마나 차이 나는지
- 단위를 그대로 유지한 채 계산한 값

예를 들어 `Ni MAE = 0.709 g/L`라면,

- 니켈 후액 농도를 예측할 때
- 평균적으로 약 `0.709 g/L` 정도 차이가 난다는 뜻이다.

즉,

- **작을수록 좋다**
- `0`이면 완벽하게 맞는 것이다.

### 0.2 raw-feed basis란?

여기서 `raw-feed basis`는 **사용자가 원래 넣는 원수(feed) 기준으로 계산하는 방식**을 말한다.

즉,

- 금속 농도는 NaOH를 섞기 전 원래 feed 농도
- NaOH는 별도의 유량/농도로 따로 넣어서
- 엔진이 내부에서 희석과 pH 변화를 직접 계산하는 방식이다.

반대로 `legacy premixed basis`는,

- NaOH가 이미 섞인 것처럼 희석된 농도와 합산 유량을 먼저 만든 뒤
- 그 값을 시뮬레이터 입력으로 넣는 방식이다.

### 0.3 random multistage stress란?

이건 현장 실험 데이터를 다시 돌려보는 테스트가 아니라,

- 유량
- pH
- stage 수
- 추출제
- 금속 농도
- 황산염 농도

같은 조건을 여러 가지로 무작위 생성해서,

**엔진이 이상한 값 없이 안정적으로 끝까지 계산되는지 보는 강건성 테스트**다.

즉,

- “실제 현장 데이터에 잘 맞는가?”를 보는 테스트가 아니라
- “이 계산 엔진이 넓은 조건에서도 잘 버티는가?”를 보는 테스트다.

### 0.4 실패했다는 말의 뜻

여기서 `실패`는 “앱이 꺼졌다”는 뜻이 아니다.

다음 중 하나가 나오면 실패로 봤다.

1. 반복 계산이 끝까지 수렴하지 않음
2. 물질수지가 크게 어긋남
3. 음수 농도처럼 물리적으로 말이 안 되는 값이 나옴

즉 `12 / 24 실패`라는 말은,

- 무작위 다단 케이스 24개 중
- 12개가 위 기준 중 하나를 만족해서
- “이 조건에서는 결과를 그대로 믿기 어렵다”고 판정됐다는 뜻이다.

### 0.5 1원리 모델이 아니라는 말

`1원리 모델`은 가능한 한 실제 화학평형과 열역학을 직접 사용해서 계산하는 모델을 말한다.

현재 모델은 그런 완전한 열역학 해석기보다는,

- 문헌값
- 경험식
- 현장 보정값
- 경량화된 경쟁추출/종분화 보정

을 섞어서 만든 실용형 모델에 가깝다.

즉,

- **빠르고 실용적이지만**
- **모든 조건에서 자동으로 일반화되는 모델은 아직 아니다**

라고 이해하면 된다.

## 1. 현재 시뮬레이터가 잘하는 것

### 1.1 현장 보정 범위에서의 다단 예측

현재 버전은 `legacy premixed basis` 기준 현장 데이터(Data1~6)에서 안정적으로 동작한다.

회귀 기준 결과:

- 아래 MAE는 “실제값과 평균적으로 얼마나 차이 나는지”를 뜻한다.
- 값이 작을수록 좋다.
- `Li MAE = 0.816 g/L`
- `Ni MAE = 0.709 g/L`
- `Co MAE = 0.060 g/L`

dataset별 MAE:

- `Data1 = 0.299 g/L`
- `Data2 = 0.164 g/L`
- `Data3 = 0.110 g/L`
- `Data4 = 0.372 g/L`
- `Data5 = 0.049 g/L`
- `Data6 = 0.255 g/L`

즉, **현재 현장 보정 범위 안에서는 후액 경향과 상대 비교를 보는 용도로 충분히 실무적**이다.

### 1.2 단일단 계산 안정성

단일단 모델은 넓은 조건에서 수치적으로 매우 안정적이다.

검증 결과:

- deterministic single-stage 대표 케이스 `6/6 PASS`
- random single-stage stress `60/60 PASS`
- 물질수지 실패 없음
- 비정상값 없음

즉, **단일 mixer-settler stage 거동 해석은 신뢰도가 높다**.

### 1.3 기본 공정 해석 기능

이 앱은 단순 결과 숫자만 내는 수준이 아니라, 공정 해석 도구로서 다음을 잘 수행한다.

- pH isotherm
- 금속별 추출률 비교
- McCabe-Thiele 해석
- 다단 counter-current profile
- NaOH profile
- field-calibrated vs raw-feed 비교

즉, **공정 조건 비교와 경향 해석 도구로서의 가치는 이미 충분하다**.

### 1.4 구조적 정합성

기본 구조 검증은 통과한다.

- single-stage mass balance PASS
- multistage mass balance PASS
- target pH self-consistent dilution PASS

즉, **숫자는 나오는데 금속 총량이 앞뒤가 안 맞는 식의 잘못된 계산은 아니라는 뜻**이다.

쉽게 말하면:

- 입력으로 들어간 금속 총량과
- 계산 후 수계 + 유기계로 나간 금속 총량이
- 거의 같게 유지된다는 뜻이다.

## 2. 현재 한계

### 2.1 범용 다단 robust simulation은 아직 아님

쉽게 말하면, **아무 조건이나 넣어도 항상 안정적으로 계산이 끝나는 수준은 아직 아니라는 뜻**이다.

즉 현장 보정 범위에서는 잘 되지만,

- stage 수가 달라지거나
- pH가 높아지거나
- 황산염이 높아지거나
- 특정 추출제 조합이 들어가면

일부 조건에서는 계산이 불안정해질 수 있다.

random multistage stress 결과:

- 이것은 무작위로 만든 다단 조건 24개를 넣어본 강건성 테스트 결과다.
- 초기 baseline 실패 `12 / 24`
- 다단 솔버 안정화 후 실패 `10 / 24`

즉 일부 개선은 했지만, **임의의 다단 운전영역 전체를 robust하게 커버한다고 말하기는 어렵다**.

### 2.2 남은 실패가 특정 조건에 집중됨

현재 남은 실패는 거의 아래 조건에 몰려 있다.

- `D2EHPA`
- `target pH`
- `고황산염`
- `상대적으로 높은 target pH`
- `다단`

즉 문제는 전체 모델이 전반적으로 무너지는 것이 아니라, **특정 운전영역에서의 취약성**으로 압축된다.

### 2.3 raw-feed 기준 검증은 아직 참고용

현재 raw-feed basis는 엔진상 계산은 가능하지만, 정식 회귀 기준으로 쓰기엔 부족하다.

쉽게 말하면:

- “원래 feed 농도 + 실제 NaOH 조건” 기준으로 계산은 할 수 있지만
- 그 계산이 맞는지 강하게 검증할 만큼 현장 메타데이터가 아직 충분하지 않다는 뜻이다.

현재 raw-feed 결과:

- `Li MAE = 0.619 g/L`
- `Ni MAE = 2.261 g/L`
- `Co MAE = 0.086 g/L`

이 차이는 모델 문제도 일부 있겠지만, 더 큰 이유는 현재 Data1~6에 실제 NaOH 농도/주입 방식 정보가 없기 때문이다.

즉, **raw-feed basis는 아직 “정식 검증 기준”이 아니라 참고용 계산 결과**다.

### 2.4 화학모델은 경량화되어 있음

현재 모델은 완전한 열역학 해석기가 아니라 다음 성격에 가깝다.

- 시그모이드 기반 준경험 모델
- 경쟁추출 보정
- loading 보정
- sulfate speciation 일부 반영
- 현장 보정 파라미터 포함

따라서 현장 범위 안에서는 빠르고 유용하지만, **모든 운전조건에서 자동으로 정확해지는 완전한 화학평형 모델은 아니다**.

이 부분을 더 보완하려면 보통 아래가 필요하다.

1. 더 많은 현장/실험 데이터
2. NaOH 농도와 주입 방식 같은 운전 메타데이터
3. sulfate complexation, activity, ionic strength 같은 열역학 정보
4. 어떤 운전영역을 공식 적용 범위로 볼지에 대한 연구원 판단

## 3. 보완을 위해 필요한 것

### 3.1 연구원에게 필요한 정보

다음 정보가 있으면 raw-feed 검증과 모델 일반화를 크게 개선할 수 있다.

1. Data1~6의 실제 NaOH 농도
2. NaOH 주입 방식
   - 전체 혼합
   - stage별 분할
   - 특정 front stage 집중
3. 입력 분석값이 raw feed인지, 이미 희석 반영값인지
4. pH 측정 기준
   - 각 stage 목표치
   - 마지막 stage 기준
   - 제어 기준점
5. 현장에서 가장 중요한 KPI
   - Li 후액
   - Ni loss
   - Co breakthrough
   - Mn complete extraction

### 3.2 연구원 판단이 필요한 것

다음은 단순 코딩 문제가 아니라 모델 철학과 적용 범위를 정하는 문제다.

1. 이 앱을 `범용 예측기`로 갈지, `현장 보정형 예측기`로 유지할지 **==> 범용 예측기로,,,**
2. D2EHPA 고황산염 조건을 실제 적용 범위로 볼지 ==> 실제 적용 범위임. (물에 대한 황산염 용해도를 고려해야 함.), 황산염 기준 맥스 2.5M
3. sulfate/activity/ionic strength 보정을 어디까지 넣을지 ==> 제한없이 그냥 적용해
4. 문헌값과 현장 보정값을 어디까지 분리할지 ==> 문헌값과 현장 보정값 기준을 구분 (현재 적용되어 있음)
5. 적용 가능 운전영역을 명시할지 (시뮬레이션 가능한 범위를 명시 필요)

### 3.3 1원리 모델에 더 가깝게 가려면 필요한 것

현재 한계를 더 줄이려면 대체로 아래가 필요하다.

1. 더 넓은 조건의 단일단 실험 데이터
   - pH
   - O/A
   - sulfate
   - extractant 농도
   - 온도
2. 다단 실증 데이터
   - stage별 pH
   - stage별 유기상/수계 분석값
   - NaOH 실제 농도와 주입 방식
3. 열역학 보강
   - sulfate complexation 강화 ==> 문헌을 찾아서 진행 검토
   - activity / ionic strength 보정 ==> 문헌 찾아서 진행 검토
   - 금속별 loading 거동 세분화 ==> 문헌 찾아서 진행 검토
4. 적용 범위 정의
   - 어디까지를 공식 예측 범위로 볼지 먼저 합의해야 함 ==> 그런건 없음.

### 3.4 실제 공정 구성 정보가 왜 중요한가

현재 남아 있는 오차 중 일부는 “모델이 틀려서”라기보다, **실제 공정을 어떻게 운전했는지 정보가 부족해서** 생기는 오차일 가능성이 크다.

쉽게 말하면, 같은 feed 농도라도 아래 정보가 다르면 결과가 달라질 수 있다.

- NaOH 농도
- NaOH 주입 위치/방식
- 실제 희석 여부
- 각 stage pH 또는 제어 기준
- 실제 유량과 O/A
- stage별 농도 변화

즉, 공정 구성 정보를 더 알면 단순히 파라미터를 억지로 맞추는 것보다 훨씬 자연스럽게 정확도를 높일 수 있다.

특히 현재 모델에서 가장 효과가 클 것으로 보이는 정보는 아래와 같다.

1. 실제 `NaOH 농도`
2. `NaOH 주입 방식`
   - 전체 혼합인지
   - stage별 분할인지
   - 앞단 집중인지
3. 입력 분석값이 `원수(raw feed)` 기준인지, 이미 희석된 값인지
4. 실제 `stage별 pH` 또는 제어 기준점
5. 실제 `O/A`, 유량 변동, 온도
6. 가능하면 `stage별 raffinate / organic 분석값`

현재 판단으로는, **정확도를 더 높이기 위해 가장 먼저 필요한 것은 파라미터 추가 보정보다 실제 공정 구성 정보**다.

## 4. 내가 이미 적용한 부분

연구원 판단 없이도 먼저 개선 가능한 영역은 이미 일부 진행했다.

### 4.1 검증 체계 강화

- legacy basis 회귀 테스트 고정
- 기준을 벗어나면 자동 FAIL
- field 성능 저하를 자동 차단

### 4.2 target pH self-consistent dilution

- target pH 모드에서 NaOH 희석 경로 지원
- raw-feed 검증 기반 마련

### 4.3 다단 솔버 안정화

- adaptive relaxation
- best-iterate 선택
- final exact-pass refinement
- target pH continuation fallback

효과:

- field 정확도 유지
- random multistage failure 일부 감소
- Cyanex 계열 취약성 감소

즉, **수치적 안정화는 먼저 적용했고, 남은 문제는 점점 도메인 판단 영역으로 수렴하고 있다**.

## 5. 연구원과 논의해야 할 핵심 질문

### A. 적용 범위 정의

- 이 시뮬레이터는 어디까지를 공식 적용 범위로 봐야 하는가
- D2EHPA 고황산염 target pH 다단 조건도 실제로 다뤄야 하는가

### B. 데이터 정의

- Data1~6은 raw-feed 기준인가, premixed 기준인가 ==> Raw-feed 값임.
- NaOH 농도와 주입 방식은 무엇인가 ==> 주입방식 : 사포니피케이션, NaOH 농도 : Cyanex272 기준은 10wt.%, D2EHPA 기준 30wt.%

### C. 정확도 우선순위

- 무엇을 가장 잘 맞춰야 하는가 ==> 추출 후액의 금속 농도 및 금속별 추출율
- 모든 금속 평균 오차보다 특정 금속 breakthrough가 더 중요한가 ==> 둘다 중요

### D. 모델 방향

- 더 복잡한 화학모델로 갈 가치가 있는가 ==> 정확도가 높아진다면,,,
- 아니면 현재처럼 경량 모델 + 현장 보정이 더 실용적인가 ==> 지금 모델이 너무 단순하다. 현실적이게 모델 수정이 필요

## 6. 실무적 제안

현재 기준으로는 다음처럼 운영하는 것이 안전하다.

### 지금 바로 가능한 사용 방식

- 현장 보정 범위 안에서 조건 비교
- 추출제/유량/pH/stage 수 sensitivity 분석
- 후액 경향 예측
- 실험/현장 데이터 비교 보조

### 지금 바로 하면 위험한 사용 방식

- 임의 고황산염 다단 조건의 절대 예측을 그대로 의사결정에 사용
- raw-feed basis 결과를 실제값처럼 단정
- 적용 범위 정의 없이 범용 공정 해석기로 사용하는 것

## 7. 현재 권장 결론

이 시뮬레이터는 현재 **현장 보정형 SX 시뮬레이터**로 보는 것이 가장 정확하다.

즉,

- 보정된 조건 범위에서는 충분히 유용하고
- 단일단 및 현장 유사 다단 해석은 신뢰할 수 있지만
- 범용 다단 robust simulator라고 단정하기는 아직 이르다.

다음 단계는 코드만 더 고치는 것보다, 먼저 연구원과 아래를 합의하는 것이 맞다.

1. 적용 범위
2. NaOH/운전 메타데이터
3. 우선 KPI
4. 일반화 수준
