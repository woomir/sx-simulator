# v2.1.1 연구원 논의자료

## 문서 목적

이 문서는 현재 SX 시뮬레이터의 상태를 연구원과 빠르게 공유하고, 다음 개선 단계에서 어떤 부분이 코드로 해결 가능한 영역인지, 어떤 부분은 도메인 판단과 현장 정보가 필요한지 정리하기 위한 회의용 자료다.

핵심 질문은 세 가지다.

1. 이 시뮬레이터가 현재 잘하는 것은 무엇인가
2. 어디까지를 현재 신뢰 가능한 적용 범위로 볼 것인가
3. 남은 한계를 줄이기 위해 무엇이 더 필요한가

---

## 1. 현재 시뮬레이터가 잘하는 것

### 1.1 현장 보정 범위에서의 다단 예측

현재 버전은 `legacy premixed basis` 기준 현장 데이터(Data1~6)에서 안정적으로 동작한다.

회귀 기준 결과:

- `Li MAE = 0.816 g/L`
- `Ni MAE = 0.709 g/L`
- `Co MAE = 0.060 g/L`

dataset별 MAE:

- `Data1 = 0.299 g/L`
- `Data2 = 0.164 g/L`
- `Data3 = 0.110 g/L`
- `Data4 = 0.372 g/L`
- `Data5 = 0.049 g/L`
- `Data6 = 0.255 g/L`

즉, **현재 현장 보정 범위 안에서는 후액 경향과 상대 비교를 보는 용도로 충분히 실무적**이다.

### 1.2 단일단 계산 안정성

단일단 모델은 넓은 조건에서 수치적으로 매우 안정적이다.

검증 결과:

- deterministic single-stage 대표 케이스 `6/6 PASS`
- random single-stage stress `60/60 PASS`
- 물질수지 실패 없음
- 비정상값 없음

즉, **단일 mixer-settler stage 거동 해석은 신뢰도가 높다**.

### 1.3 기본 공정 해석 기능

이 앱은 단순 결과 숫자만 내는 수준이 아니라, 공정 해석 도구로서 다음을 잘 수행한다.

- pH isotherm
- 금속별 추출률 비교
- McCabe-Thiele 해석
- 다단 counter-current profile
- NaOH profile
- field-calibrated vs raw-feed 비교

즉, **공정 조건 비교와 경향 해석 도구로서의 가치는 이미 충분하다**.

### 1.4 구조적 정합성

기본 구조 검증은 통과한다.

- single-stage mass balance PASS
- multistage mass balance PASS
- target pH self-consistent dilution PASS

즉, **코드가 계산은 되는데 물질수지가 틀리는 형태는 아니다**.

## 2. 현재 한계

### 2.1 범용 다단 robust simulation은 아직 아님

임의 운전조건 전체에서 다단이 항상 안정적으로 풀리는 수준은 아니다.

random multistage stress 결과:

- 초기 baseline 실패 `12 / 24`
- 다단 솔버 안정화 후 실패 `10 / 24`

즉 일부 개선은 했지만, **임의의 다단 운전영역 전체를 robust하게 커버한다고 말하기는 어렵다**.

### 2.2 남은 실패가 특정 조건에 집중됨

현재 남은 실패는 거의 아래 조건에 몰려 있다.

- `D2EHPA`
- `target pH`
- `고황산염`
- `상대적으로 높은 target pH`
- `다단`

즉 문제는 전체 모델이 전반적으로 무너지는 것이 아니라, **특정 운전영역에서의 취약성**으로 압축된다.

### 2.3 raw-feed 기준 검증은 아직 참고용

현재 raw-feed basis는 엔진상 계산은 가능하지만, 정식 회귀 기준으로 쓰기엔 부족하다.

현재 raw-feed 결과:

- `Li MAE = 0.619 g/L`
- `Ni MAE = 2.261 g/L`
- `Co MAE = 0.086 g/L`

이 차이는 모델 문제도 일부 있겠지만, 더 큰 이유는 현재 Data1~6에 실제 NaOH 농도/주입 방식 정보가 없기 때문이다.

즉, **raw-feed basis는 아직 “정식 검증 축”이 아니라 진단용 preview**다.

### 2.4 화학모델은 경량화되어 있음

현재 모델은 완전한 열역학 해석기가 아니라 다음 성격에 가깝다.

- 시그모이드 기반 준경험 모델
- 경쟁추출 보정
- loading 보정
- sulfate speciation 일부 반영
- 현장 보정 파라미터 포함

따라서 현장 범위 안에서는 빠르고 유용하지만, **모든 운전조건에서 일반화되는 1원리 모델은 아니다**.

## 3. 보완을 위해 필요한 것

### 3.1 연구원에게 필요한 정보

다음 정보가 있으면 raw-feed 검증과 모델 일반화를 크게 개선할 수 있다.

1. Data1~6의 실제 NaOH 농도
2. NaOH 주입 방식
   - 전체 혼합
   - stage별 분할
   - 특정 front stage 집중
3. 입력 분석값이 raw feed인지, 이미 희석 반영값인지
4. pH 측정 기준
   - 각 stage 목표치
   - 마지막 stage 기준
   - 제어 기준점
5. 현장에서 가장 중요한 KPI
   - Li 후액
   - Ni loss
   - Co breakthrough
   - Mn complete extraction

### 3.2 연구원 판단이 필요한 것

다음은 단순 코딩 문제가 아니라 모델 철학과 적용 범위를 정하는 문제다.

1. 이 앱을 `범용 예측기`로 갈지, `현장 보정형 예측기`로 유지할지
2. D2EHPA 고황산염 조건을 실제 적용 범위로 볼지
3. sulfate/activity/ionic strength 보정을 어디까지 넣을지
4. 문헌값과 현장 보정값을 어디까지 분리할지
5. 적용 가능 운전영역을 명시할지

## 4. 내가 이미 적용한 부분

연구원 판단 없이도 먼저 개선 가능한 영역은 이미 일부 진행했다.

### 4.1 검증 체계 강화

- legacy basis 회귀 테스트 고정
- 기준을 벗어나면 자동 FAIL
- field 성능 저하를 자동 차단

### 4.2 target pH self-consistent dilution

- target pH 모드에서 NaOH 희석 경로 지원
- raw-feed 검증 기반 마련

### 4.3 다단 솔버 안정화

- adaptive relaxation
- best-iterate 선택
- final exact-pass refinement
- target pH continuation fallback

효과:

- field 정확도 유지
- random multistage failure 일부 감소
- Cyanex 계열 취약성 감소

즉, **수치적 안정화는 먼저 적용했고, 남은 문제는 점점 도메인 판단 영역으로 수렴하고 있다**.

## 5. 연구원과 논의해야 할 핵심 질문

### A. 적용 범위 정의

- 이 시뮬레이터는 어디까지를 공식 적용 범위로 봐야 하는가
- D2EHPA 고황산염 target pH 다단 조건도 실제로 다뤄야 하는가

### B. 데이터 정의

- Data1~6은 raw-feed 기준인가, premixed 기준인가
- NaOH 농도와 주입 방식은 무엇인가

### C. 정확도 우선순위

- 무엇을 가장 잘 맞춰야 하는가
- 모든 금속 평균 오차보다 특정 금속 breakthrough가 더 중요한가

### D. 모델 방향

- 더 복잡한 화학모델로 갈 가치가 있는가
- 아니면 현재처럼 경량 모델 + 현장 보정이 더 실용적인가

---

## 6. 실무적 제안

현재 기준으로는 다음처럼 운영하는 것이 안전하다.

### 지금 바로 가능한 사용 방식

- 현장 보정 범위 안에서 조건 비교
- 추출제/유량/pH/stage 수 sensitivity 분석
- 후액 경향 예측
- 실험/현장 데이터 비교 보조

### 지금 바로 하면 위험한 사용 방식

- 임의 고황산염 다단 조건의 절대 예측을 그대로 의사결정에 사용
- raw-feed basis 결과를 실제값처럼 단정
- 적용 범위 정의 없이 범용 공정 해석기로 사용하는 것

---

## 7. 현재 권장 결론

이 시뮬레이터는 현재 **현장 보정형 SX 시뮬레이터**로 보는 것이 가장 정확하다.

즉,

- 보정된 조건 범위에서는 충분히 유용하고
- 단일단 및 현장 유사 다단 해석은 신뢰할 수 있지만
- 범용 다단 robust simulator라고 단정하기는 아직 이르다.

다음 단계는 코드만 더 고치는 것보다, 먼저 연구원과 아래를 합의하는 것이 맞다.

1. 적용 범위
2. NaOH/운전 메타데이터
3. 우선 KPI
4. 일반화 수준

---

## 8. 회의 후 바로 실행 가능한 후보

연구원과 논의가 끝나면 아래 둘 중 하나로 바로 이어갈 수 있다.

### 후보 1. Raw-feed 검증 승격

- 실제 NaOH 농도/주입 방식 반영
- raw-feed basis를 공식 회귀 기준으로 승격

### 후보 2. D2EHPA 고황산염 일반화

- 남아 있는 다단 취약 조건 집중 개선
- 단, 적용 범위와 모델 철학을 먼저 합의해야 함
