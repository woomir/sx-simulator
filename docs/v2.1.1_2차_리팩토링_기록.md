# v2.1.1 2차 리팩토링 기록

작성일: 2026-03-08

## 목적

2차 리팩토링의 목적은 Streamlit 세션에서 조정한 추출제 파라미터가 엔진 전역 상수에 직접 반영되던 구조를 분리하는 것이다.

기존 구조의 문제점:

- `sx_dashboard.py`가 `EXTRACTANT_PARAMS` 전역 딕셔너리를 직접 수정했다.
- 같은 프로세스 안에서 세션 간 상태 오염 가능성이 있었다.
- 검증 스크립트와 비교 탭이 숨은 전역 상태에 의존할 수 있었다.
- 테스트 순서에 따라 결과가 달라질 위험이 있었다.

이번 리팩토링은 이 결합도를 줄이되, 사용자 기능과 시뮬레이션 결과는 바꾸지 않는 것을 목표로 했다.

## 변경 범위

### 1. 엔진 함수가 파라미터 세트를 명시적으로 받도록 변경

다음 함수들에 `extractant_params=None` 인자를 추가했다.

- `sx_simulator/extraction_isotherm.py`
  - `get_effective_pH50`
  - `get_effective_k`
  - `extraction_efficiency`
  - `distribution_coefficient`
  - `calc_loading_fraction`
  - `calc_free_NaL`
  - `compute_competitive_extractions`
  - `get_proton_release`
  - `compute_all_extractions`
  - `print_isotherm_table`
- `sx_simulator/single_stage.py`
  - `_partition_stage_with_damping`
  - `_solve_competitive_stage_state`
  - `solve_single_stage`
  - `_solve_at_target_pH`
  - `_solve_with_fixed_NaOH`
- `sx_simulator/multistage_sx.py`
  - `solve_multistage_countercurrent`

기본값은 `None`으로 두었고, 인자가 생략되면 기존 전역 설정을 그대로 사용하도록 유지했다. 따라서 대시보드 외부에서 엔진을 호출하는 기존 코드와의 호환성은 유지된다.

### 2. 대시보드가 전역 파라미터를 수정하지 않도록 변경

`sx_dashboard.py`에서 다음 구조 변경을 적용했다.

- `apply_params_to_global()` 제거
- `get_active_extractant_params()` 추가
- 세션 상태 `st.session_state.custom_params`를 현재 활성 파라미터 세트의 단일 출처로 사용
- 메인 계산, 비교 탭 계산, 등온선/분배계수/수식 탭 계산이 모두 `extractant_params=active_extractant_params`를 명시적으로 전달
- 파라미터 데이터베이스 탭도 현재 세션 파라미터를 기준으로 표를 생성

이제 대시보드 내부 파라미터 변경은 현재 사용자 세션에만 영향을 준다.

### 3. 피팅 탭의 파라미터 적용 대상을 세션 상태로 제한

기존:

- 피팅 결과 적용 버튼이 `EXTRACTANT_PARAMS[extractant][metal]`을 직접 수정

변경 후:

- 피팅 결과 적용 버튼이 `st.session_state.custom_params[extractant][metal]`만 수정

이 변경으로 피팅 후 즉시 현재 세션 시뮬레이션에는 반영되지만, 엔진 전역 상수는 오염되지 않는다.

## 동작 보존 원칙

이번 리팩토링에서 의도적으로 바꾸지 않은 항목:

- 기본 추출식과 보정식
- 금속 경쟁 추출 로직
- 종분화(speciation) 로직
- 다단 역류 수렴 알고리즘
- UI 흐름과 탭 구성
- 기존 검증 데이터와 기대값

즉, 계산 구조의 "입력 전달 방식"만 바꾸고, 화학 모델과 수치 알고리즘은 바꾸지 않았다.

## 회귀 검증 결과

다음 검증을 다시 수행했다.

### 1. 문법 검증

실행:

```bash
python3 -m py_compile sx_dashboard.py test_verification.py sx_simulator/datasets.py sx_simulator/single_stage.py sx_simulator/extraction_isotherm.py sx_simulator/multistage_sx.py
```

결과:

- PASS

### 2. 구조 검증

실행:

```bash
.venv/bin/python validation_test.py
```

결과:

- 단일 stage 물질수지 PASS
- Co isotherm 일관성 PASS
- 다단 역류 물질수지 PASS
- 수렴: 53 iterations

### 3. 현장 데이터 비교 검증

실행:

```bash
.venv/bin/python test_verification.py
```

결과:

- 기존과 같은 검증 흐름 유지
- 주요 오차 패턴도 기존 수준 유지
- 예시:
  - Data1 Ni: `-6.397 g/L`
  - Data2 Ni: `-2.996 g/L`
  - Data4 Li: `+2.160 g/L`
  - Data6 Co: `-0.845 g/L`

즉, 이번 리팩토링으로 정확도가 좋아지거나 나빠지지 않았고, 기존 모델의 정합도 수준이 그대로 유지됐다.

### 4. 앱 부팅 확인

실행:

```bash
.venv/bin/streamlit run sx_dashboard.py --server.headless true --server.port 8765
```

결과:

- 앱 정상 부팅 확인
- `Local URL: http://localhost:8765`

## 남은 리팩토링 후보

### 우선순위 높음

1. `sx_dashboard.py` 탭 렌더링 분리
- 현재 대시보드 파일이 너무 크다.
- 탭별 렌더러 함수 또는 모듈로 분리하면 충돌과 회귀 위험이 줄어든다.

2. 입력 상태와 계산 상태 분리
- 현재는 Streamlit 위젯 값 파싱과 시뮬레이션 실행이 한 파일에서 강하게 결합돼 있다.
- `build_simulation_inputs()` 같은 계층을 만들면 검증과 테스트가 쉬워진다.

3. 파라미터 세트 자료형 명확화
- 현재는 중첩 딕셔너리다.
- 추후 `dataclass` 또는 타입 힌트 기반 구조체로 바꾸면 안정성이 더 올라간다.

### 우선순위 중간

1. 탭 내부 중복 chart/helper 정리
- 미니 isotherm, 표 생성, metric 포맷팅 등이 반복된다.

2. 검증 스크립트의 자동 assert 강화
- 지금은 보고서형 출력 비중이 크다.
- 허용오차 기준을 둔 자동 실패 조건을 추가할 수 있다.

3. 파라미터 피팅 결과의 export/import 분리
- 현재는 세션 반영 중심이다.
- 추후 JSON profile 저장/불러오기 기능으로 확장 가능하다.

## 참고

따라서 `CHANGELOG.md`에는 반영하지 않았다.
