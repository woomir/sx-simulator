# v2.1.1 4차 리팩토링 기록

작성일: 2026-03-08

## 목적

4차 리팩토링의 목적은 `sx_dashboard.py` 안에 직접 하드코딩돼 있던 긴 정적 설명 콘텐츠를 외부 markdown 문서로 분리하는 것이다.

이번 단계는 특히 다음 문제를 줄이기 위해 진행했다.

- 설명 문구 수정이 코드 수정과 배포 이력에 섞이던 구조
- 대시보드 파일이 UI 로직과 대량의 정적 텍스트를 동시에 포함하던 구조
- 사용자 문서성 콘텐츠가 코드 리뷰 diff를 불필요하게 크게 만들던 문제

## 변경 파일

- `sx_dashboard.py`

## 주요 변경 내용

### 1. markdown 로더 helper 추가

`sx_dashboard.py`에 `load_markdown_doc()` helper를 추가했다.

역할:

- `docs/` 또는 루트의 markdown 문서를 읽어 반환
- 문서가 없을 때 기본 메시지 표시

이 helper로 정적 탭 콘텐츠를 코드에서 분리하면서도 기존 렌더 흐름은 유지했다.

### 2. 정적 탭 콘텐츠 외부 문서화

다음 탭 콘텐츠를 별도 markdown 파일로 옮겼다.

- 용어 해설 탭 -> `docs/glossary.md`
- 참고 문헌 서술부 -> `docs/references.md`
- 데이터 피팅 검증 이력 탭 -> `docs/validation_history.md`

이제 설명 텍스트 수정은 문서 파일만 바꾸면 된다.

### 3. 기존 문서 기반 탭과 구조 일관화

이미 외부 문서를 읽고 있던 `사용자 매뉴얼` 탭과 같은 패턴으로 정리했다.

결과적으로 현재 정적 성격이 강한 탭들은 다음처럼 문서 기반으로 동작한다.

- 사용자 매뉴얼
- 용어 해설
- 참고 문헌 서술부
- 데이터 피팅 검증 이력
- 변경 이력

### 4. 구조 문서 보강

루트의 `PROJECT_STRUCTURE.md`에 새로 분리된 사용자 문서 파일들을 반영했다.

## 동작 보존 원칙

이번 리팩토링은 정적 콘텐츠의 저장 위치만 바꾸는 작업이다.

바꾸지 않은 항목:

- 시뮬레이션 계산 로직
- 파라미터
- 엔진 수치해법
- 사용자 입력 구조
- 결과 계산 및 검증 흐름

즉, 웹앱의 계산 결과와 시뮬레이션 능력에는 영향이 없다.
