# v2.1.2 알칼리 정합성 구현 및 회귀 결과

작성일: 2026-03-11

이 문서는 `v2.1.2_알칼리_정합성_인터페이스_초안.md`를 실제 코드에 반영한 내용과, 반영 후 field 회귀 결과를 기록한다.

---

## 1. 이번 반영 범위

### 1.1 인터페이스/해석 정리

- `sx_simulator/alkali_contract.py`를 추가해 알칼리 경로를 명시적으로 분리했다.
  - `aqueous_direct`
  - `fresh_organic_saponification`
- 사포니피케이션 경로는 두 모델을 명시적으로 구분한다.
  - `physical_v2`
  - `legacy_equivalent_target`
- `fresh organic saponification + target pH`는 기본적으로 **stage별 pH 제어**가 아니라 **raffinate pH 목표**로 해석하도록 바꿨다.
- legacy equivalent-target 경로는 숨겨진 기본값이 아니라 **명시적 fallback**으로만 남겼다.

### 1.2 엔진 구현

- 다단 사포니피케이션에서 `fresh sap inventory`가 실제로 organic side를 따라 역류 전달되도록 구현했다.
- stage 결과에 아래 진단값을 추가했다.
  - `aqueous_naoh_in_mol_hr`
  - `aqueous_naoh_effective_mol_hr`
  - `saponified_extractant_in_mol_hr`
  - `saponified_extractant_out_mol_hr`
  - `saponification_direct_neutralization_mol_hr`
  - `saponification_metal_exchange_mol_hr`
  - `saponification_inventory_error_mol_hr`
- 결과 payload에 아래 메타데이터를 추가했다.
  - `alkali_application`
  - `alkali_control_mode`
  - `alkali_target_kind`
  - `saponification_model`
  - `legacy_fallback_used`

### 1.3 UI 반영

- Streamlit UI에서 사포니피케이션 + 목표 모드의 기본 문맥을 `후액 목표 pH`로 바꿨다.
- 사포니피케이션에서는 `Stage별 차등 pH`를 기본 흐름에서 제거했다.
- 필요한 경우에만 `Legacy equivalent-target fallback`을 켤 수 있게 했다.

### 1.4 파라미터 미세 보정

남아 있던 상대오차를 줄이기 위해 field-calibrated override를 추가했다.

- `Cyanex 272 / Li`
  - `pH50 = 8.00 -> 8.10`
  - `k = 2.0 -> 1.8`
- `D2EHPA / Li`
  - `pH50 = 6.50 -> 6.30`
- `D2EHPA / Co`
  - `E_max = 99.5 -> 99.0`

## 3. field 회귀 결과

정확도 해석 기준:

- 기본: 상대오차 `%`
- trace 영역(`exp < 0.1 g/L`): 절대오차 `g/L` 보조 해석

### 3.1 Cyanex 272

aggregate relative error:

- `legacy_premixed_target_pH = 13.8%`
- `raw_feed_target_pH = 46.9%`
- `raw_feed_fixed_saponification = 23.4%`
- `raw_feed_physical_saponification_v2 = 167.3%`

주요 금속 relative error:

- Li: `10.4% / 39.1% / 31.3% / 65.1%`
- Ni: `18.8% / 58.5% / 11.5% / 320.5%`

판단:

- legacy release gate와 raw-feed legacy fallback은 소폭 개선됐다.
- `physical_v2`는 구현 자체는 되었지만, 현재 파라미터/화학모델 상태로는 Cyanex 272 field를 재현하지 못한다.
- 특히 Ni extraction이 과도하게 약해져 아직 승격 불가다.

### 3.2 D2EHPA

aggregate relative error:

- `legacy_premixed_target_pH = 25.3%`
- `raw_feed_target_pH = 65.8%`
- `raw_feed_fixed_saponification = 50.8%`
- `raw_feed_physical_saponification_v2 = 1686.3%`

주요 금속 relative error:

- Li: `22.9% / 38.7% / 38.5% / 40.6%`

판단:

- legacy release gate는 유지하면서 Li/Co가 소폭 개선됐다.
- `physical_v2`는 D2EHPA에서 특히 Co/Ni 재현성이 크게 무너져 experimental 단계에 머문다.

  - `fresh_saponification_target_physical`: convergence `2/12`
  - `fresh_saponification_target_legacy`: convergence `12/12`

- 따라서 이 경로를 기본 UI에서 제거하고 explicit legacy fallback으로만 남긴 판단은 유지한다.

## 4. 구현 후 결론

### 4.1 이번에 확인된 것

- 인터페이스 분리와 legacy fallback의 explicit화는 완료했다.
- `fresh sap inventory`가 다단에서 실제로 역류 전달되는 물리 경로는 구현했다.
- 기존 release gate(`legacy_premixed_target_pH`)는 유지됐다.
- 남아 있던 `Cyanex Li`, `D2EHPA Li/Co`는 소폭이지만 줄였다.

### 4.2 아직 미해결인 것

- `raw_feed_physical_saponification_v2`는 아직 field 정확도 기준을 만족하지 못한다.
- 현재 `physical_v2`는 두 용매 모두 extractant-specific chemistry가 부족하다.
  - Cyanex 272: sap-conditioned Ni extraction이 너무 약함
  - D2EHPA: Co/Ni under-extraction과 pH 재현성 문제가 큼

### 4.3 운영 판단

- `legacy_premixed_target_pH`: 계속 release gate 유지
- `raw_feed_fixed_saponification`: legacy shadow reference 유지
- `raw_feed_physical_saponification_v2`: 구현 완료, 그러나 **experimental/shadow only**

즉 이번 v2.1.2에서는

1. 인터페이스/알칼리 해석을 분리했고
2. physical sap cascade를 실제로 넣었으며
3. field 회귀를 다시 돌렸고
4. 남는 Li/Co 오차를 소폭 보정했다.

하지만 `physical_v2` 자체는 아직 승격 조건을 충족하지 못하므로, 다음 단계에서는 sap-conditioned extraction chemistry를 extractant별로 더 보강해야 한다.
