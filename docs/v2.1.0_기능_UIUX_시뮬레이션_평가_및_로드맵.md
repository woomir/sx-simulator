# v2.1.0 기능/UI·UX/시뮬레이션 평가 및 로드맵

작성일: 2026-03-07

## 1. 평가 목적

이 문서는 현재 웹앱이 실제로 "용매추출(SX) 시뮬레이터"라고 부를 수 있는 수준인지 판단하고,
어떤 보완이 즉시 필요하며 중장기적으로 무엇을 더 강화해야 하는지 정리하기 위한 검토 보고서다.

이번 평가는 아래 3축으로 나누어 수행했다.

- 기능적 관점: 웹앱이 공정 조건 입력, 계산, 결과 해석, 비교 분석을 일관되게 제공하는가
- UI/UX 관점: 사용자가 결과를 오해하지 않고 안전하게 해석할 수 있도록 안내하는가
- 시뮬레이션 관점: 물질수지, 다단 수렴, 현장 데이터 정합성, 일반화 가능성이 충분한가

검토에 사용한 근거는 다음과 같다.

- `validation_test.py` 재실행
- `test_verification.py` 재실행
- Streamlit 대시보드 기동 확인
- `sx_dashboard.py`, `sx_simulator/config.py`, `test_verification.py` 코드 리뷰

## 2. 총평

현재 웹앱은 "다단 용매추출 공정의 경향성과 상대 비교를 해석하는 SX 시뮬레이터"로는 충분히 성립한다.
단일단 및 다단 물질수지가 맞고, 다단 역류 계산도 안정적으로 수렴하며, pH/단수/OA/추출제 농도 변화에 대한 공정 응답을 비교하는 데 실질적으로 사용할 수 있다.

다만 아직 "현장 농도를 범용적으로 정량 예측하는 고신뢰 디지털 트윈" 수준은 아니다.
현장 검증 데이터 6세트 기준으로 일부 조건은 잘 맞지만, 특정 Ni/Co/Li 조건에서 오차가 크다.
따라서 현재 권장 포지션은 아래와 같다.

- 적합: 내부 공정 비교, 경향 분석, 민감도 검토, 프리테스트 설계
- 주의 필요: 절대 농도 예측, 현장 설비 설정값 직접 결정, 검증 범위 밖 조건 적용

## 3. 이번에 반영한 즉시 개선 사항

이번 검토 과정에서 아래 개선을 코드에 반영했다.

### 3.1 기능 측면

- 현장 보정값과 문헌 기본값을 구분한 파라미터 프로필 선택 기능 추가
- 검증 스크립트에서 대시보드와 동일한 황산염 계산식을 사용하도록 정렬
- 검증 스크립트에 데이터셋별 MAE, 최악 오차 금속, 금속별 전체 MAE 요약 추가

### 3.2 UI/UX 측면

- 사이드바와 결과 탭에 "검증 커버리지" 안내 추가
- 결과 탭에 수렴 여부와 반복 횟수 표시 추가
- Data5 Li 데이터 이상치에 대한 경고를 프리셋 UI와 검증 스크립트에 명시
- 비교 탭에서 현재 비교 조건(C_ext, 단수, 온도, 황산염)을 직접 표시

### 3.3 시뮬레이션 신뢰성 측면

- 현장 검증 범위를 벗어난 입력에 대해 사용자가 범위를 인지할 수 있도록 해석 가이드 추가
- 현장 보정 파라미터와 문헌 기준선을 분리해, "현장 맞춤"과 "범용 비교"를 구분해서 해석할 수 있도록 개선

## 4. 기능적 평가

### 4.1 현재 상태 평가

기능적으로는 이미 SX 시뮬레이터로 부를 수 있다.
사용자는 피드 조성, 추출제 종류, 추출제 농도, 단수, pH 전략, NaOH 공급 방식, 온도를 입력할 수 있고,
결과는 후액 농도, 추출률, stage별 거동, isotherm, McCabe-Thiele, 추출제 비교 등으로 이어진다.
단순 계산기가 아니라 공정 해석 툴로서 필요한 기본 골격은 갖추고 있다.

좋은 점은 다음과 같다.

- 단일 결과만 보여주지 않고 다단 해석과 비교 분석 흐름까지 포함함
- 프리셋과 사용자 입력을 함께 지원함
- 파라미터 수정과 데이터 피팅 기능이 연결되어 있음
- 계산 엔진과 대시보드가 분리되어 있어 구조가 비교적 명확함

남아 있는 약점은 다음과 같다.

- 검증 통과 여부가 아직 "자동 판정"보다는 "출력 확인"에 가까움
- 계산 결과를 저장하거나 시나리오를 비교 이력으로 관리하는 기능이 없음
- 결과를 보고서 형태로 내보내는 기능이 없음
- 입력 조합이 많아 실무 사용자가 같은 조건을 반복 재현하기 어렵다

### 4.2 단기 보완 권고

- 검증 스크립트에 허용오차 기반 PASS/FAIL 기준 추가
- 시나리오 저장/불러오기 기능 추가
- 주요 입력과 결과를 CSV 또는 PDF로 내보내는 기능 추가
- 수렴 실패 시 재시도 가이드 또는 권장 조정 항목 표시

### 4.3 중장기 보완 권고

- 조건 스윕(batch run) 기능 추가
- 조건별 결과를 누적 저장하는 실험 로그 기능 추가
- 현장 배치별 보정 프로필 관리 기능 추가
- API 또는 내부 서비스 형태로 엔진을 분리해 다른 시스템과 연동 가능하게 개선

## 5. UI/UX 평가

### 5.1 현재 상태 평가

UI는 정보량이 많고 분석 탭도 충분하지만, 아직 "전문가용 툴"에 가깝다.
즉, 알고 있는 사용자는 유용하게 쓸 수 있지만, 처음 보는 사용자는 어떤 결과를 얼마나 믿어야 하는지 바로 파악하기 어렵다.

이번 수정으로 다음 부분은 개선됐다.

- 활성 보정 프로필이 무엇인지 명확히 표시됨
- 현재 조건이 현장 검증 범위와 얼마나 겹치는지 직접 노출됨
- 수렴 여부와 반복 횟수가 드러나 계산 상태를 숨기지 않음
- Data5 Li처럼 해석하면 안 되는 데이터 포인트가 UI에서 경고됨

그래도 남아 있는 UX 리스크는 있다.

- 사이드바 입력 항목이 많아 처음 사용자에게 부담이 큼
- 결과 탭이 많아 어디부터 봐야 하는지 학습이 필요함
- "문헌 기본값"과 "현장 보정"의 차이를 모르는 사용자는 결과를 혼용할 수 있음
- 수치가 많지만 권장 해석 순서가 아직 약함

### 5.2 단기 보완 권고

- 입력 영역을 `기본`, `고급`, `보정`, `검증` 섹션으로 더 명확히 구분
- 결과 상단에 `먼저 볼 것` 카드 추가
- preset 선택 시 해당 데이터의 용도와 한계를 한 줄로 안내
- 색상 대비와 강조 체계를 단순화해 경고/정상/참고 메시지 구분 강화

### 5.3 중장기 보완 권고

- "빠른 검토 모드"와 "전문가 모드" 분리
- 입력-계산-검증-결과 해석의 step-by-step 워크플로우 제공
- 결과 비교를 위한 스냅샷 또는 diff UI 추가
- 모바일/저해상도 환경에서의 가독성 개선

## 6. 시뮬레이션 평가

### 6.1 현재 상태 평가

시뮬레이션 엔진의 구조적 타당성은 좋다.
단일단 물질수지와 다단 역류 물질수지는 재검증 시 모두 통과했고,
앱도 정상적으로 기동했다.

검증 결과 요약은 다음과 같다.

- `validation_test.py`: PASS
- 다단 예시 케이스: 수렴 성공, Co 물질수지 오차 매우 작음
- `test_verification.py`: 모든 데이터셋 수렴
- 그러나 현장 정합도는 일부 케이스에서 여전히 큼

재검증 기준 주요 오차는 다음과 같다.

- Data1 Ni: 약 -6.397 g/L
- Data2 Ni: 약 -2.996 g/L
- Data4 Li: 약 +2.160 g/L
- Data6 Co: 약 -0.845 g/L

금속별 평균 절대오차는 다음 수준이다.

- Li: 약 0.826 g/L
- Ni: 약 1.943 g/L
- Co: 약 0.206 g/L

즉, 이 엔진은 "계산이 안 되는 모델"이 아니라 "구조는 맞지만 일부 구간에서 현장 정합성이 제한된 모델"이다.

### 6.2 핵심 우려 사항

가장 큰 우려는 일반화 가능성이다.
현재 엔진은 준경험(sigmoid + competition + speciation light) 기반이므로,
현장 보정값이 들어간 상태에서 다른 현장/희석제/이온강도 조건까지 그대로 일반화된다고 보기는 어렵다.

구체적인 리스크는 다음과 같다.

- Data1~6 중심 보정이 다른 현장까지 그대로 통할 보장은 없음
- 고이온강도/고로딩 구간에서 활동도와 비이상성이 충분히 반영되지 않음
- 유기상 거동은 여전히 휴리스틱 비중이 높음
- entrainment, phase disengagement, crud, 실제 mixer-settler 비이상성은 미포함
- 검증 데이터 수가 아직 적고, 단수/온도/희석제 범위가 좁음

### 6.3 단기 보완 권고

- 글로벌 파라미터와 사이트별 보정 파라미터 분리
- 검증용 데이터와 피팅용 데이터 분리
- 데이터셋별 허용오차 기준 설정
- 현장 범위 밖 조건에 대한 경고를 더 강하게 표시
- known bad datapoint를 코드와 문서에서 일관되게 관리

### 6.4 중기 보완 권고

- 단일단 isotherm 데이터와 다단 공정 데이터를 분리해 각각 피팅/검증
- hold-out 또는 cross-validation 방식 도입
- 금속별, 추출제별, 조건별 오차 리포트 자동 생성
- pH, O/A, C_ext, 총금속농도, 황산염에 대한 민감도 지도 구축

### 6.5 장기 보완 권고

- 활동도 계수 기반의 비이상성 모델 도입 검토
- 유기상 loading/complexation 모델의 물리화
- uncertainty quantification 도입
- 파라미터 역추정(fitting) 파이프라인 자동화
- 실험실 단일단 데이터와 파일럿/현장 다단 데이터를 함께 쓰는 계층형 보정 체계 구축

## 7. 실행 권고안

현재 버전은 아래 용도로는 바로 사용 가능하다.

- 조건 탐색
- 공정 비교
- stage 수와 pH 전략의 상대 비교
- 내부 엔지니어링 검토

아래 용도로는 추가 보완 후 사용하는 편이 안전하다.

- 현장 설비 설정값 직접 결정
- 신규 현장 조건에 대한 절대 농도 예측
- 검증 범위를 벗어난 고이온강도/고온/단수 변화 조건의 정량 해석

권장 운영 방식은 다음과 같다.

- 기본값은 `현장 보정 (Isd 108)` 프로필로 두되, 비교 시 `문헌 기본값`도 함께 확인
- 결과를 볼 때는 `수렴 상태`, `검증 커버리지`, `최대 로딩률`을 먼저 확인
- 검증 범위 밖 조건은 "예측"이 아니라 "방향성 참고"로 해석

## 8. 결론

현재 웹앱은 용매추출 시뮬레이터라고 부를 수 있다.
다만 그 의미는 "현장 범위 내 공정 해석용 시뮬레이터"에 가깝고,
"모든 조건에서 그대로 믿을 수 있는 범용 예측기"까지는 아니다.

이번 수정으로 사용자는 이제 다음을 더 명확히 볼 수 있게 됐다.

- 어떤 파라미터 기준으로 계산했는지
- 현재 조건이 검증 범위 안인지
- 계산이 실제로 수렴했는지
- 검증 데이터 중 무엇이 신뢰하기 어려운지

다음 우선순위는 명확하다.

1. 검증 자동화 강화
2. 글로벌/현장 보정 파라미터 분리
3. 오차 지도의 체계적 축적
4. 비이상성 및 일반화 성능 강화
