# v2.1.2 알칼리 정합성 방향성 검증

검증일: 2026-03-11

이 문서는 `v2.1.2_알칼리_정합성_인터페이스_초안.md`의 방향이 실제 문제를 맞게 겨냥하는지, **구현 전 엔진 기준**으로 확인한 결과를 정리한 것이다.

구현 후 실제 반영 결과와 재회귀 수치는 `docs/v2.1.2_알칼리_정합성_구현_및_회귀.md`에 기록했다.

주의:

- 당시에는 `physical_saponification_v2`가 아직 구현되지 않았었다.
- 따라서 이 문서는 “새 초안이 해결하려는 문제가 실제로 존재하는가”, “우선순위가 맞는가”를 보는 **사전 방향성 검증** 문서로 해석해야 한다.

---

## 1. 검증 방법

실행 커맨드:

```bash
python3 tests/validate_v212_direction.py
```

검증 축:

1. **현장 데이터 검증**
   - 용매별로 `legacy_premixed_target_pH`
   - `raw_feed_target_pH`
   - `raw_feed_fixed_saponification`
   를 비교
2. **랜덤 데이터 검증**
   - 용매별로 mode별 convergence / mass balance / runtime 비교
3. **핵심 방향성 체크**
   - 현재 `target pH + saponification` 경로가 `Q_NaOH` 변화에 실제로 반응하는지 확인

---

## 2. 현장 데이터 결과

이 문서에서는 정확도를 다음처럼 해석한다.

- 기본: `|시뮬레이션 - 실험| / |실험| × 100` 상대오차 `%`
- 단, 실험 후액값이 아주 작은 trace 영역(`예: < 0.1 g/L`)은 상대오차가 불안정하므로 절대오차 `g/L`를 함께 본다.

### 2.1 Cyanex 272

aggregate relative error:

- `legacy_premixed_target_pH = 14.2%`
- `raw_feed_target_pH = 48.1%`
- `raw_feed_fixed_saponification = 23.7%`

주요 금속 relative error:

- Li: `11.1% / 40.7% / 31.9%`
- Ni: `18.9% / 59.2% / 11.4%`
- Co: trace 영역이 많아 `%` 지표보다는 절대오차 해석이 더 적절

판단:

- `raw_feed_target_pH`보다 `raw_feed_fixed_saponification`이 훨씬 낫다.
- 특히 Cyanex 272에서는 **Ni 오차가 크게 줄어든다**.
- 즉 fresh organic 사포니피케이션을 `stage별 pH 제어`보다 **sap inventory 문제로 보는 방향이 타당**하다.

### 2.2 D2EHPA

aggregate relative error:

- `legacy_premixed_target_pH = 27.1%`
- `raw_feed_target_pH = 66.5%`
- `raw_feed_fixed_saponification = 51.0%`

주요 금속 relative error:

- Li: `23.5% / 39.4% / 39.2%`
- Ni: `2.2% / 60.1% / 10.1%`
- Co: `54.5% / 91.0% / 99.8%`

판단:

- D2EHPA에서도 `raw_feed_fixed_saponification`이 `raw_feed_target_pH`보다 훨씬 낫다.
- 특히 **Ni 오차가 극적으로 줄어든다**.
- 다만 D2EHPA는 여전히 legacy보다 오차가 크므로, 새 물리 경로를 넣더라도 **즉시 legacy를 대체하면 안 된다**.

### 2.3 field 결과 종합 판단

- 두 용매 모두 `raw_feed_target_pH`보다 `raw_feed_fixed_saponification`이 우세했다.
- 따라서 초안의 핵심 주장:
  - `fresh organic saponification`은 `stage별 pH 제어 수단`이 아니다.
  - `equivalent target pH`를 즉시 제거하지 말고 legacy fallback으로 남겨야 한다.
  는 현재 데이터와 정합적이다.
- 다만 상대오차 기준으로 보면 `raw_feed_fixed_saponification`도 Li/Co에서 여전히 큰 편차가 남고, D2EHPA 전체 상대오차도 아직 크다.
- 따라서 초안 구현만으로 곧바로 “현장 정확도 충분” 상태가 되지는 않는다.

## 3. 랜덤 데이터 결과

### 3.1 Cyanex 272

- `aqueous_direct_fixed`: convergence `1/12`, balance fail `11`
- `aqueous_direct_target`: convergence `12/12`, balance fail `0`
- `fresh_saponification_fixed`: convergence `12/12`, balance fail `0`
- `fresh_saponification_target_legacy`: convergence `12/12`, balance fail `0`

해석:

- 이번 랜덤 범위에서는 `fresh_saponification_fixed`가 매우 안정적이었다.
- `aqueous_direct_fixed`는 상대적으로 취약했다.
- 즉 새 인터페이스 수용 기준에 **mode별 random stress 분리 집계**를 넣은 것은 맞는 방향이다.

### 3.2 D2EHPA

- `aqueous_direct_fixed`: convergence `7/12`, balance fail `6`
- `aqueous_direct_target`: convergence `12/12`, balance fail `1`
- `fresh_saponification_fixed`: convergence `12/12`, balance fail `1`
- `fresh_saponification_target_legacy`: convergence `12/12`, balance fail `3`

해석:

- D2EHPA는 Cyanex 272보다 random 조건에서 더 민감했다.
- 같은 용매추출 시뮬레이터라도 **용매별 승격 기준을 분리해야 한다**는 초안 보완안이 타당하다.
- `fresh_saponification_target_legacy`는 수렴은 되지만 `fresh_saponification_fixed`보다 mass balance fail이 더 많았다.

### 3.3 random 결과 종합 판단

- `fixed_input`과 `solve_to_target`, `aqueous_direct`와 `fresh_saponification`을 한데 묶어 회귀로 보면 실제 취약성이 가려진다.
- 따라서 초안에 추가한
  - 용매별 집계
  - mode별 convergence / balance / runtime 집계
  - release gate와 shadow diagnostic 분리
  는 실제로 필요하다.

## 4. 핵심 확인: target pH + saponification의 Q_NaOH 무감도

랜덤 12건씩 확인 결과:

- Cyanex 272: invariant `12/12`
- D2EHPA: invariant `12/12`
- 최대 raffinate 차이: `0.000000000000 g/L`
- 최대 pH profile 차이: `0.000000000000`

의미:

- 현재 multistage `target pH + saponification` 경로는 `Q_NaOH`를 바꿔도 결과가 변하지 않았다.
- 즉 사용자가 `Q_NaOH`를 입력해도 실제 다단 결과에 반영되지 않는다는 초안의 문제 인식이 **사실상 그대로 확인**됐다.

판단:

- 이 조합을 기본 UI에서 허용하는 것은 부적절하다.
- 초안대로 `target pH + saponification`은 기본적으로 `raffinate pH target`으로만 제한하고,
- 기존 경로는 legacy fallback 또는 명시적 호환 모드로만 남기는 것이 맞다.

## 5. 초안만으로 정확도 향상이 충분한가

결론부터 말하면 **아니다**.

이 초안은 정확도 향상에 필요한 기반을 바로잡는 문서이지, 그 자체로 모든 상대오차를 줄여주는 문서는 아니다.

### 5.1 이 초안으로 기대할 수 있는 개선

- 물리적으로 잘못된 제어 조합 제거
- `Q_NaOH` 무감도 같은 구조적 오류 제거
- raw-feed와 legacy replay를 분리한 검증 체계 확보
- fresh sap inventory를 실제 counter-current state로 반영할 수 있는 구조 확보

즉 **알칼리 해석 오류 때문에 생기던 정확도 손실**은 줄일 수 있다.

### 5.2 이 초안만으로 해결되지 않는 부분

- 실측 NaOH 농도/주입 위치 메타데이터 부족
- D2EHPA의 Co, Li 잔여 상대오차
- Cyanex 272의 Li 잔여 상대오차
- 경쟁추출/종분화/파라미터 모델 자체의 잔여 편차

즉 구현 후에도 상대오차 `%`를 더 낮추려면,

1. 새 인터페이스와 물리 경로 구현
2. 상대오차 `%` 기준 field 회귀 재검증
3. 남는 금속·용매 조합에 대한 파라미터/모델 보정

이 추가로 필요하다.

## 6. 최종 판단

이번 검증 기준으로 보면 초안의 방향은 전반적으로 맞다.

### 유지해야 할 방향

- `fresh organic saponification`을 `sap inventory` 문제로 재해석
- `target pH + saponification` 기본 노출 제거
- `legacy fallback` 즉시 제거 금지
- 용매별 승격 기준 분리
- field + random + sensitivity를 함께 보는 회귀 체계 도입

### 아직 보류해야 할 것

- `raw_feed`를 즉시 1차 pass/fail 기준으로 승격
- `equivalent target pH` 경로의 조기 제거
- 단일 aggregate 지표만 보고 새 경로를 승격하는 결정

### 실무적 결론

- 초안의 **문제 정의와 우선순위는 타당**하다.
- 다만 구현 후 승격은
  - `legacy_premixed_target_pH`
  - `raw_feed_fixed_saponification_legacy`
  - `raw_feed_physical_saponification_v2`
  병행 검증을 거친 뒤 결정해야 한다.
- 다시 말해, **초안은 필요하지만 충분조건은 아니다**.
