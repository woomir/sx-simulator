# v2.1.1 다음 단계 연구원 협의 기획

## 목적

현재 시뮬레이터는 구조 안정화, field-calibrated 회귀 보호, raw-feed basis 진단 경로까지는 확보된 상태다.

추가로 8단계에서 다단 솔버 수치 안정화는 먼저 적용했다. 따라서 이 문서에 남아 있는 항목은, 이제 **단순 코드 정리나 수치해석 개선만으로는 결정할 수 없는 도메인 판단 영역**이라고 보면 된다.

다만 다음 단계는 단순 구현 문제가 아니라 도메인 판단이 필요한 영역이라, 연구원 협의 없이 바로 진행하면 잘못된 가정을 코드에 고정할 위험이 있다.

이 문서는 바로 실행하기 위한 작업지시서가 아니라, **연구원과 먼저 합의해야 할 의사결정 항목**을 정리한 기획 문서다.

---

## 트랙 A. Raw-Feed 기준 검증 승격

### 목표

현재 공식 회귀 기준인 `legacy_premixed_target_pH`를 장기적으로 `raw_feed_target_pH` 기반 검증으로 전환할 수 있는지 판단한다.

즉,

- 원수 농도(raw feed)
- 실제 NaOH 유량
- 실제 NaOH 농도
- 필요하면 stage별 주입 방식

을 기준으로 엔진을 검증하는 체계로 옮기는 것이 목표다.

### 왜 연구원 협의가 필요한가

현재 Data1~6에는 `naoh_flow`는 있지만, 실제 NaOH 농도나 stage별 운전 방식 정보가 없다.

그래서 raw-feed basis를 돌릴 수는 있지만, 지금은 `NaOH 5.0 M` 가정에 의존한다.

이 상태에서 raw-feed를 공식 pass/fail 기준으로 올리면:

- 모델 자체 오차
- 잘못된 NaOH 가정 오차

가 섞여서 판단이 왜곡될 수 있다.

### 연구원에게 확인해야 할 항목

1. Data1~6의 실제 NaOH 농도는 얼마인가
2. NaOH는 전체 mixer-settler train에 균일하게 투입된 것인가, stage별 분할 투입인가
3. `feed_flow`와 `naoh_flow`는 계측 기준이 무엇인가
4. 검증표의 입력 농도는 원수 기준인가, 이미 일부 희석이 반영된 분석값인가
5. pH는 각 stage 목표치인가, 최종 후단 제어 기준인가

### 협의 후 구현 가능한 작업

1. `datasets.py`에 dataset별 `naoh_concentration_m` 추가
2. 필요 시 dataset별 `naoh_distribution_strategy` 추가
3. `prepare_verification_case()`가 실측 NaOH 정보를 우선 사용하도록 전환
4. `test_verification.py`에서 raw-feed 기준 허용오차 재설계
5. 일정 조건 충족 시 raw-feed를 공식 회귀 기준으로 승격

### 성공 기준

- raw-feed 기준에서 `listed NaOH flow`와 `estimated NaOH flow`의 괴리가 작아짐
- raw-feed 기준 MAE가 legacy 기준과 동등하거나 더 우수
- dataset별 오차 패턴이 물리적으로 더 납득 가능

### 현재 판단

이 트랙은 **데이터 메타정보가 확보되기 전까지는 보류**가 맞다.

## 트랙 B. 시뮬레이션 정확도 추가 개선

### 목표

현 시점에서 남아 있는 대표 오차를 더 줄여, 시뮬레이터의 정량 예측 성능을 높인다.

현재 남아 있는 대표 패턴:

- D2EHPA 고황산염 조건의 Li/Ni 잔여 오차
- raw-feed basis에서 Ni 오차 확대
- 경쟁추출/고로딩 조건에서의 조건 의존적 편차

### 왜 연구원 협의가 필요한가

이 트랙은 단순 수치 튜닝이 아니라 화학적 의미를 건드린다.

예를 들면:

- sulfate complexation을 D 계산에 어느 수준까지 반영할지
- free metal 기준으로 경쟁추출 우선순위를 바꿀지
- 금속별 loading penalty를 अलग अलग 둘지
- activity / ionic strength 보정을 넣을지

이런 선택은 정확도뿐 아니라 모델의 해석 가능성과 일반화 범위에 직접 영향을 준다.

즉, 연구원 의견 없이 진행하면 “현재 현장만 잘 맞는 보정”으로 흐를 위험이 있다.

### 연구원에게 확인해야 할 항목

1. 실제 공정/실험에서 sulfate complexation 영향이 큰 금속 우선순위는 무엇인가
2. 현장에서 중요하게 보는 KPI는 무엇인가
   - 후액 Li
   - Ni loss
   - Co breakthrough
   - Mn 완전 제거
3. 어떤 조건 범위를 이 앱의 공식 적용 범위로 볼 것인가
4. 문헌 기반 기본값과 현장 보정값을 어디까지 분리할 것인가
5. 활동도/이온강도 보정이 필요한 수준인지, 아니면 현재 경량 모델 유지가 더 중요한지

### 협의 후 구현 가능한 작업

1. free-metal fraction을 D 계산에 더 직접 반영
2. 경쟁추출 우선순위를 조건 의존형으로 전환
3. 금속별/추출제별 loading penalty 세분화
4. 필요 시 activity correction의 경량 버전 추가
5. 개선 후 legacy + raw-feed 동시 회귀 검증

### 성공 기준

- legacy regression thresholds를 안정적으로 유지
- raw-feed basis 성능도 동시에 악화되지 않음
- 특정 dataset 한두 개만 좋아지는 로컬 fit이 아니라 전체 패턴이 개선됨

### 현재 판단

이 트랙은 **연구원과 “무엇을 물리적으로 반영할지”를 먼저 합의한 뒤 착수**하는 편이 맞다.
