# v2.1.1 시뮬레이션 검증 누적 로그

이 문서는 시뮬레이션 엔진 변경 단계마다 수행한 검증 결과를 누적 기록한다.

검증 원칙:

- 구조 변경 단계라도 검증을 생략하지 않는다.
- 수치 결과가 의도적으로 변하지 않아야 하는 단계는 이전 패턴 유지 여부를 기록한다.
- 구조 검증과 현장 비교 검증을 분리해서 기록한다.

## 2단계 검증. 황산염 착물/자유 금속 speciation 연결

### 검증 목적

`K_MSO4`가 실제 speciation 계산에 연결되었는지 확인하고, 기존 시뮬레이션 결과를 불필요하게 깨지 않는지 확인한다.

### 수행 항목

1. 문법/구문 검증
2. speciation helper 직접 호출 확인
3. `validation_test.py`
4. `test_verification.py`

### 결과

#### 1. 문법/구문 검증

- `compile()` 기반 syntax check 통과
- 대상 파일 6개 모두 오류 없음

#### 2. speciation helper 직접 확인

조건:

- 금속: `Co`
- `pH = 5.5`
- `C_sulfate = 1.5 M`

결과:

- `free_metal_fraction = 0.002903`
- `hydroxo_fraction = 0.000000` 수준
- `sulfate_fraction = 0.997097`
- `free_sulfate_m = 1.499537`

해석:

- 고황산염 조건에서 Co는 대부분 sulfate complex 형태로 존재하도록 계산된다.
- 따라서 hydrolysis 기여가 매우 작아지고, `K_MSO4`가 실제 speciation 계산에 반영됨을 확인했다.

#### 3. `validation_test.py`

- Single stage mass balance: PASS
- Co isotherm consistency: PASS
- Multistage counter-current logic: PASS
- 다단 예제 수렴: `53 iterations`
- Co cascade balance diff: 약 `0.000050 g`

#### 4. `test_verification.py`

주요 후액 오차 패턴은 이전 단계와 동일하게 유지됐다.

- `Data1 Ni`: `-6.397 g/L`
- `Data2 Ni`: `-2.996 g/L`
- `Data4 Li`: `+2.160 g/L`
- `Data6 Co`: `-0.845 g/L`

금속별 MAE 요약도 유지:

- `Li`: `0.826 g/L`
- `Ni`: `1.943 g/L`
- `Co`: `0.206 g/L`
- `Mn`: `0.000 g/L`
- `Ca`: `0.000 g/L`
- `Mg`: `0.000 g/L`
- `Zn`: `0.000 g/L`

### 결론

2단계는 의도대로 `K_MSO4`를 실제 speciation 계산에 연결했고, 기존 현장 비교 패턴은 깨지지 않았다.

다만 이번 단계는 `pH/speciation submodel` 개선에 가깝기 때문에, 후액 농도 정확도 자체는 아직 크게 달라지지 않았다.

즉 다음 단계에서는 `free metal`을 추출 계산 쪽에 어떻게 연결할지 더 직접적으로 다뤄야 한다.

## 3단계 검증. D2EHPA-Co 고황산염 상대 보정

### 검증 목적

고황산염 D2EHPA 조건에서 Co 과추출을 줄이되, Ni나 전체 수지 검증을 망치지 않는지 확인한다.

### 수행 항목

1. 문법/구문 검증
2. sulfate D 보정 계수 직접 확인
3. `validation_test.py`
4. `test_verification.py`

### 결과

#### 1. 문법/구문 검증

- `Cyanex 272 / Co / sulfate 1.5 M`: `1.0`
- `D2EHPA / Ni / sulfate 1.5 M`: `1.0`

해석:

- 이번 단계 보정은 의도대로 `D2EHPA-Co` 조합에만 작동한다.
- 검증 범위의 저/중 sulfate 조건에서는 거의 영향이 없고, 고황산염 쪽만 선택적으로 억제한다.

#### 3. `validation_test.py`

#### 4. `test_verification.py`

변화가 거의 없는 항목:

### 결론

3단계는 broad sulfate correction이 아니라 `D2EHPA-Co`에 한정한 선택 보정으로 정리했고, 그 결과 검증상 유의미한 개선을 확인했다.

특히 Co MAE가 크게 줄었고, 고황산염 D2EHPA 조건에서 Ni도 일부 동반 개선됐다.

다만 Cyanex 272 조건의 저 pH Ni 과추출 문제는 그대로 남아 있으므로, 다음 단계는 Ni 쪽 현장 재보정 또는 목표 pH 구조 보정으로 이어가는 편이 맞다.

## 5단계 검증. 목표 pH 모드 NaOH 희석 self-consistent 계산

### 검증 목적

선택적으로 추가한 `target pH + C_NaOH` 경로가 아래 두 조건을 만족하는지 확인한다.

1. 새 경로를 사용할 때 `fixed NaOH` 계산과 수치적으로 일관되어야 한다.
2. 기존 field verification 흐름은 기본 OFF 상태에서 그대로 유지되어야 한다.

### 수행 항목

1. 문법/구문 검증
2. `validation_test.py` 신규 synthetic consistency check 포함 실행
3. `test_verification.py` 실행
4. Streamlit 앱 부팅 확인

### 결과

#### 1. 문법/구문 검증

- `compile()` 기반 syntax check 통과
- 대상 파일 8개 모두 오류 없음

#### 2. `validation_test.py`

기존 Step 1~3은 그대로 PASS.

신규 `Step 4: Target pH Self-Consistent Dilution Check` 결과:

- 가정 `NaOH 농도 = 5.0 M`
- 자동 역산 `NaOH 유량 = 4.0427 L/hr`
- target 모드 희석 반복 횟수: `15`
- 동일 유량을 `fixed NaOH`에 재주입했을 때:
  - `pH_out = 4.5001`
  - 목표 pH `4.5000`
  - 차이 `0.000100`

target-vs-fixed 후액 차이:

- `Li`: `0.000000 g/L`
- `Ni`: `0.000004 g/L`
- `Co`: `0.000067 g/L`
- `Mn`: `0.000011 g/L`

즉 새 target pH 희석 경로는 고정 NaOH 계산과 사실상 동일한 해를 재현했다.

추가 multistage smoke check:

- 3단 예제 수렴: `49 iterations`
- `NaOH_flow_profile = [3.9227, 0.4128, 0.1428] L/hr`
- `pH_profile = [4.5, 4.5, 4.5]`

즉 stage별 자동 NaOH 유량이 다음 stage 수계 유량에 정상 누적되는 것도 확인했다.

#### 3. `test_verification.py`

이번 단계는 기본 동작을 유지하도록 설계했기 때문에 field verification 결과는 4단계와 동일하게 유지됐다.

주요 항목:

- `Data1 Ni`: `-1.945 g/L`
- `Data2 Ni`: `+0.954 g/L`
- `Data3 Ni`: `-0.044 g/L`
- `Data4 Li`: `+2.143 g/L`
- `Data6 Co`: `-0.070 g/L`

금속별 MAE:

- `Li`: `0.816 g/L`
- `Ni`: `0.709 g/L`
- `Co`: `0.060 g/L`
- `Mn`: `0.000 g/L`
- `Ca`: `0.000 g/L`
- `Mg`: `0.000 g/L`
- `Zn`: `0.000 g/L`

#### 4. Streamlit 부팅 확인

- `http://localhost:8765` 정상 기동 확인

### 결론

5단계는 field MAE를 직접 바꾸는 단계가 아니라, 목표 pH 모드에 선택적 희석 self-consistent 계산 경로를 추가한 단계다.

검증상 새 경로는 `fixed NaOH` 해와 정합적으로 동작했고, 기존 현장 검증 패턴도 유지됐다.

즉 이번 단계는 **정확도 개선을 위한 기반 확장**으로 판단할 수 있다.
