# SX 시뮬레이터 v1.7.0 종합 검증 계획

## 검증 목적

v1.7.0에서 추가된 Phase 3 기능(추출제 경쟁, 수계 종분화)을 포함하여, **기존 기능과의 상호작용까지 포함한 전 조건 교차 검증**을 실시합니다.

---

## 1. 검증 매트릭스 (조건 조합)

시뮬레이터의 핵심 분기는 아래 **4가지 독립 축**으로 구성됩니다.

| 축               | 옵션 A              | 옵션 B       | 비고                                              |
| ---------------- | ------------------- | ------------ | ------------------------------------------------- |
| **pH 제어 모드** | 목표 pH (자동 NaOH) | 고정 NaOH    | `_solve_at_target_pH` vs `_solve_with_fixed_NaOH` |
| **추출제 경쟁**  | OFF (기존 sigmoid)  | ON (Phase 3) | `use_competition`                                 |
| **수계 종분화**  | OFF                 | ON (Phase 3) | `use_speciation`                                  |
| **추출제 종류**  | Cyanex 272          | D2EHPA       | 파라미터 셋 다름                                  |

> [!IMPORTANT]
> 총 **2 × 2 × 2 × 2 = 16가지** 조합이 존재합니다. 각 조합에서 에러 없이 결과가 출력되고, 값이 물리적으로 타당한지 확인합니다.

---

## 2. 검증 항목별 체크리스트

### 2-A. 시뮬레이션 정상 실행 (Runtime)

각 16가지 조합에 대해:

- [ ] **에러 없이 실행 완료** (TypeError, ZeroDivisionError 등 없음)
- [ ] **수렴 확인** (`converged == True`)
- [ ] **결과 딕셔너리 키 완전성** (`C_aq_out`, `C_org_out`, `pH_out`, `extraction`, `loading_fraction` 등 누락 없음)

### 2-B. 물질수지 (Mass Balance) 정합성

각 금속에 대해:

```
C_aq_in × Q_aq + C_org_in × Q_org ≈ C_aq_out × Q_aq_eff + C_org_out × Q_org
```

- [ ] **단일 Stage** 기준 물질수지 오차 < 0.1%
- [ ] **다단(4-stage) 기준** — Feed 금속량 = 후액 금속량 + Loaded Organic 금속량 (전체 수지)
- [ ] **NaOH 희석 효과** (고정 NaOH 모드: `Q_aq_eff = Q_aq + Q_NaOH`) 정합성

### 2-C. pH 수지 (Proton Balance) 정합성

- [ ] **목표 pH 모드**: 출구 pH가 설정값과 일치 (오차 < 0.01)
- [ ] **고정 NaOH 모드**: NaOH 투입량에 비례하여 pH 상승
- [ ] **`calc_aq_protons()` 함수**: speciation ON/OFF에 따라 H⁺ 계산값 차이 발생 확인
- [ ] **Saponification 소비**: Cyanex 272에서 NaOH 소비에 사포닌화분이 반영되는지 확인

### 2-D. Phase 3: 추출제 경쟁 모델 (Competition)

- [ ] **경쟁 OFF vs ON 비교**: 낮은 로딩(< 20%)에서는 거의 차이 없음
- [ ] **고로딩 조건 테스트**: 추출제 농도를 낮추거나(0.1M) 금속 농도를 높여(Ni 30 g/L) 로딩률 > 80% 시나리오
  - 경쟁 ON 시 후순위 금속(Ni, Li)의 추출률 감소 확인
  - `HA_free_fraction`이 0에 가깝게 떨어지는지 확인
- [ ] **추출 우선순위**: Mn > Co > Ni > Li 순서로 추출제 소비 확인
- [ ] **D_adjusted < D_sigmoid**: 경쟁 보정이 항상 D를 감소시키는 방향인지 확인

### 2-E. Phase 3: 수계 종분화 (Speciation)

- [ ] **종분화 OFF vs ON 비교**: pH 5 기준으로 NaOH 소비량 차이 확인
- [ ] **고 pH 시나리오 (pH 7+)**: 종분화 ON 시 MOH⁺ 형성으로 NaOH 소비량 증가 확인
- [ ] **`hydrolysis_H` 값**: Ni, Co가 Li보다 큰 K_MOH를 가짐 → MOH⁺ 기여가 Ni/Co에서 더 클 것
- [ ] **고정 NaOH 모드에서**: 동일 NaOH 투입 시 종분화 ON이면 출구 pH가 더 낮아야 함

### 2-F. 온도 의존성

- [ ] **25°C (기준)**: 기본값에서 정상 동작
- [ ] **40°C**: pH₅₀ 감소(추출 용이) 확인 → 동일 pH에서 추출률 증가
- [ ] **15°C**: pH₅₀ 증가(추출 어려움) 확인 → 동일 pH에서 추출률 감소
- [ ] **k 보정**: 온도에 따라 시그모이드 전이 기울기 변화 확인

### 2-G. Stage별 차등 pH / NaOH 분배

- [ ] **Stage별 차등 pH**: 4단 기준 [4.0, 4.5, 5.0, 5.5] 입력 → 각 Stage 출구 pH 일치
- [ ] **NaOH 분배 전략**: uniform vs front_loaded vs custom → 각각 pH 프로파일 차이 확인
- [ ] **커스텀 가중치**: [3, 2, 1, 0.5] 입력 시 앞단에 NaOH 집중 확인

### 2-H. UI 연동 검증

- [ ] **사이드바 위젯 변경 → 결과 즉시 반영** (모든 슬라이더, 입력값)
- [ ] **모델 수식 탭**: Phase 3 ON 시 추가 수식 표시
- [ ] **McCabe-Thiele 다이어그램**: 금속별 조작선/평형선 정상 렌더링
- [ ] **추출제 비교 탭**: Cyanex 272 vs D2EHPA 결과 차이 확인
- [ ] **데이터 피팅 탭**: CSV 업로드 → 시그모이드 피팅 정상 동작

**① 시뮬레이션 결과 탭** — `result` 딕셔너리를 직접 참조

- [ ] pH 변경(5→6) 시: 추출률, pH 프로파일, NaOH 소비 모두 변경

**④ McCabe-Thiele 탭** — `result`, `extractant`, `C_ext`, `temperature`, `Q_aq/Q_org` 참조

- [ ] A/O 비 변경(1:1→2:1) 시: 조작선 기울기(`slope = Q_aq/Q_org`) 변경
- [ ] 추출제 변경 시: 평형 곡선 형태 변경
- [ ] Stage 수 변경 시: 단수 계단선 포인트 수 변경

**⑤ 추출제 비교 탭** — 자체 `compare_pH` 슬라이더 + `C_aq_feed`, `n_stages`, `temperature` 참조

- [ ] '비교 목표 pH' 변경 시: 바 차트 추출률 값 변경
- [ ] 온도 변경 시: Isotherm 비교 곡선 이동

**⑥ 상세 데이터 탭** — `result` 딕셔너리를 직접 참조

- [ ] 시뮬레이션 조건 변경 시: 수렴 정보, Stage별 테이블, 물질수지 테이블 모두 갱신
- [ ] 물질수지 잔차 값이 0에 가까운지 확인 (< 0.01 g/hr)
- [ ] NaOH 모드 변경(목표→고정) 시: 수렴 정보와 pH 표시 변경

## 3. 검증 방법

### 자동화 테스트 스크립트 (`test_verification.py`)

Python CLI에서 16가지 조합을 자동 실행하고 결과를 표로 출력합니다:

```python
# 의사 코드
for pH_mode in ["target_pH", "fixed_NaOH"]:
    for competition in [False, True]:
        for speciation in [False, True]:
            for extractant in ["Cyanex 272", "D2EHPA"]:
                result = solve_multistage_countercurrent(...)
                assert result["converged"]
                check_mass_balance(result)
                check_pH_balance(result)
                check_extraction_range(result)
```

### 수동 브라우저 검증 (Streamlit UI)

핵심 시나리오 5~6개를 브라우저 에이전트로 스크린샷 캡처:

1. 기본 조건 (모든 OFF) → 결과 스크린샷
2. 경쟁 ON + 종분화 ON → 결과 비교
3. 고로딩 조건 (C_ext = 0.1M) → 경쟁 효과 극대화
4. 고정 NaOH 모드 → pH 프로파일 확인
5. 온도 40°C → 추출률 변화 확인

## 4. 예상 이슈 및 주의사항

| 이슈                           | 설명                                                                                 | 심각도  |
| ------------------------------ | ------------------------------------------------------------------------------------ | ------- |
| **고정 NaOH + bisection 모드** | `target_pH`가 단일값일 때 bisection으로 최적 NaOH를 찾는 로직 → NoneType 에러 가능성 | 🔴 높음 |
| **로딩률 100% 초과**           | 경쟁 모델 OFF 시 감쇠가 부족하면 물리적으로 불가능한 로딩 발생                       | 🟡 중간 |
| **Speciation 상수 K_MSO4**     | `config.py`에 정의되어 있으나 현재 `calc_aq_protons()`에서 사용되지 않음             | 🟡 중간 |
| **D2EHPA + Speciation**        | D2EHPA는 사포닌화(NaL)가 0이므로 → Speciation 효과만 독립적으로 작동해야 함          | 🟢 낮음 |
