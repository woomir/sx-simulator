# SX 시뮬레이터 실증 데이터 기반 고도화 계획 (v1.9.0)

## 1. 목표

연구원이 제공한 현장 데이터(Data1~6)를 활용하여 시뮬레이터의 **정합성을 현장(Isd 108) 환경에 맞추어 100% 가깝게 끌어올리고(Data Fitting)**, 이 데이터를 시연 및 검증 목적에 즉각 활용할 수 있도록 **"프리셋 자동 세팅 UI"를 구축**합니다.

## 2. 사용자 피드백 대기 (User Review Required)

추출제 기초 파라미터를 보정(Fitting)하는 작업이 포함되어 있습니다.

> [!IMPORTANT]
> **Data Fitting 방향성 승인 필요:**
>
> - Cyanex 272 기준, `Ni`의 $pH_{50}$ 파라미터를 소폭 우측(추출이 덜 되는 방향)으로 보정합니다. (Data1, 2 니켈 코발트 분리도 현상 반영)
> - D2EHPA 기준, `Mg`의 $pH_{50}$ 파라미터를 소폭 좌측(추출이 더 잘 되는 방향)으로 보정합니다. (Data5 마그네슘 조기 추출 현상 반영)
>   해당 보정은 연구원님의 현장 용매(Isd 108 등) 특성에 맞게 우리 모델을 '최적화'하는 방향입니다.

## 3. 세부 구현 계획 (Proposed Changes)

---

### [데이터베이스 (알고리즘 보정)]

#### [MODIFY] `sx_simulator/config.py`

- `EXTRACTANT_PARAMS` 딕셔너리 미세 보정:
  - **Cyanex 272:** `Ni`의 $pH_{50}$ 파라미터를 `5.5` 근방에서 `5.8 ~ 6.0` 수준으로 소폭 상향(추출 지연).
  - **D2EHPA:** `Mg`의 $pH_{50}$ 파라미터를 조금 더 산성 구간(좌측)으로 이동하여 pH 4 전후에서도 100% 추출이 가능하도록 보정.

---

### [UI/UX (프리셋 기능 개발)]

#### [MODIFY] `sx_dashboard.py`

- 사이드바 설정 영역의 최상단(또는 Feed 조건 위)에 **"🧪 현장 실험 Data 프리셋 (자동 입력)"** 이라는 Selectbox UI 추가.
- 옵션 리스트: `["사용자 직접 입력", "Data1 (CoSX-D9)", "Data2 (CoSX-D5)", "Data3 (CoSX-D2)", "Data4 (IMSX-D5)", "Data5 (IMSX-D9)", "Data6 (IMSX-D?)" ]`
- **Session State (Callback) 컨트롤 로직:**
  - 사용자가 특정 Data(예: Data1)를 선택하면, Streamlit의 `st.session_state`를 강제로 업데이트하여 다음 값들이 즉각 렌더링되게 구현:
    - **농도:** 각 금속 농도 자동 기입 (예: Ni 33.89, Co 3.09 ...) _희석 비율을 자동 반영할 지 논의에 따라 원액 or 희석액 입력 가능하도록 처리._
    - **유기상:** 추출제 종류(Cyanex 272/D2EHPA), 농도($C_{ext}$) 자동 기입.
    - **단수:** 5단(Mixer-Settler) 고정.
    - **O/A 비율:** $Q_{org}$ / $Q_{aq}$ 자동 계산 입력.
    - **목표 pH:** 목표 단일 pH로 세팅.
