## 2. 발견된 이슈 및 수정 사항

### 🔴 이슈 1: bisection 모드 물질수지 오류 (수정 완료)

- **원인**: 단일 `target_pH` 시 `multistage_sx.py`가 bisection으로 최적 NaOH를 찾으면서 `_solve_with_fixed_NaOH(Q_NaOH=0)`을 호출하여 물질수지가 깨짐
- **수정**: `target_pH`를 `target_pH_per_stage = [target_pH] * n_stages`로 변환하여 `_solve_at_target_pH`로 직접 계산
- **파일**: `sx_simulator/multistage_sx.py` line 62-64
- **결과**: target_pH 모드 물질수지 오차 0.000%로 개선

### 🟡 이슈 2: 고정 NaOH 모드 수렴 실패

- **현상**: 500회 반복 후에도 수렴하지 않음 (tolerance=1e-6)
- **영향**: 물질수지 오차 증가 (특히 Ni 9.6%)
- **원인 추정**: counter-current 반복에서 organic phase 값이 수렴하기 어려운 조건
- **대응**: 향후 solver 개선 검토 필요 (relaxation factor 등)

### 🟢 이슈 3: 종분화(Speciation) 효과 미미

- **현상**: ON/OFF 결과가 동일 (NaOH 25.61 mol/hr)
- **원인**: `K_MOH` 값이 10⁻¹⁰ 수준 → pH 5에서 MOH⁺ 농도 ≈ 10⁻²¹ M (물리적으로 무시 가능)
- **판정**: 코드 자체는 정상. pH > 10 이상에서만 유의미한 효과 발생
- **대응**: 사용자에게 "이 파라미터 조건에서는 종분화 효과가 경미함" 안내 필요

### 🟡 이슈 4: D2EHPA + 경쟁 모드 수렴 실패 (#6, #8)

- **현상**: D2EHPA에서 경쟁 모드 ON 시 500회 반복 후 미수렴
- **원인 추정**: D2EHPA의 pH₅₀가 Cyanex보다 낮아 금속 추출이 더 빠르게 일어남 → 경쟁 보정과 counter-current 반복 충돌
- **대응**: 향후 수렴 로직 최적화 필요
