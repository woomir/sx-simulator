# 📘 Li/Ni/Co/Mn Mixer-Settler SX 시뮬레이터 사용자 매뉴얼

본 매뉴얼은 시뮬레이터의 목적, 각 화면 요소의 기능, 그리고 결과를 안전하게 해석하기 위한 사용 원칙을 정리한 문서입니다.

---

## 1. 개요 및 활용 목적

본 시뮬레이터는 배터리 재활용 등 습식제련(Hydrometallurgy) 공정에서 여러 금속(Li, Ni, Co, Mn, Ca, Mg, Zn)이 혼합된 수용액으로부터 특정 금속을 선택적으로 분리해내는 **용매 추출(Solvent Extraction, SX)** 공정을 예측합니다.

- **활용 목적**:
  - **조업 조건 도출**: O/A 비, 단수(Stages), 초기 pH, NaOH 투입량 등 최적의 조업 조건을 현장 실험 전에 사전 시뮬레이션합니다.
  - **스케일업 및 문제 해결**: 현장에서 예상치 못하게 추출률이 떨어지는 원인(예: 추출제 부족, 종분화 완충 등)을 모델을 통해 진단합니다.
  - **추출제 성능 비교**: Cyanex 272와 D2EHPA 중 어떤 추출제가 현재 원수 비율에 적합한지 빠르게 판별합니다.

---

## 2. 화면 구성 및 핵심 조작법

화면은 크게 **좌측 사이드바(입력부)**와 **우측 메인 대시보드(결과 출력부)**로 나뉩니다.

### A. 좌측 사이드바 (입력 및 설정)

시뮬레이션 구동에 필요한 모든 환경 변수를 조작합니다.

1.  **🧪 현장 실험 Data 프리셋**
    - **가장 먼저 테스트해 볼 항목입니다.** 연구개발 그룹에서 검토한 현장 데이터(Data1~6) 환경을 원클릭으로 불러올 수 있습니다.
    - 프리셋을 변경하면 아래의 농도, 유량, 추출제, 단수, 목표 pH 설정이 그에 맞게 자동 세팅됩니다.
    - 일부 프리셋에는 데이터 품질 주의사항이 함께 표시됩니다. 예를 들어 **Data5는 Li 데이터가 물질수지상 일관되지 않으므로 Li 지표 해석에 주의**해야 합니다.
2.  **🧭 모델 보정 프로필**
    - **현장 보정 (Isd 108)**:
      - 현재 현장 프리셋(Data1~6)과 더 가깝게 맞춘 파라미터 집합입니다.
      - 현장 조건 재현, 내부 공정 비교, 상대 경향 확인에 적합합니다.
    - **문헌 기본값**:
      - 현장 맞춤 보정을 일부 되돌린 비교용 기준선입니다.
      - 특정 현장 데이터에 덜 묶인 기준을 보고 싶을 때 사용합니다.
    - 두 프로필의 결과가 크게 다르면, 현재 조건이 현장 보정에 민감한 영역일 가능성이 큽니다.
3.  **🚰 Feed (수계) 조건**
    - 반입되는 원수의 각 금속의 초기 농도(g/L)를 입력합니다.
    - 황산염($\text{SO}_4^{2-}$) 농도는 입력된 금속들을 기준으로 100% 황산염 염(Salt) 형태로 결합되어 있다고 가정하여 시스템이 **자동 연산**하므로 직접 입력하실 필요가 없습니다. (수계 버퍼 용량 계산에 사용됨)
4.  **🛢️ Organic (유기계) 조건**
    - **추출제 선택**: Cyanex 272(코발트 우선 추출)와 D2EHPA(불순물 우선 추출) 중 실험 목적에 맞는 추출제를 선택합니다.
    - **O/A 비율 (Phase Ratio)**: 유기상(Organic)과 수계상(Aqueous)의 부피 투입 비율입니다. 1.0은 동량 투입을 의미하며, 농축이 필요할 땐 0.5 근처로 낮추기도 합니다.
5.  **⚙️ 반응 설비 및 타겟 조건**
    - **Mixer-Settler 단수 (추출단)**: 실제 공정에 설치될 반응조의 개수입니다. 단수가 늘어날수록 추출 한계치에 수렴합니다.
    - **운전 온도**: 온도는 반응 속도와 화학양론에 미미하지만 영향을 줍니다.
    - **pH 제어 방식**: "고정 NaOH 투입 모드"와 "목표 pH (자동 역산)" 모델 중 하나를 택합니다. **초심자는 "목표 pH 자동 역산" 모드를 적극 권장**합니다.
    - **NaOH 적용 방식**:
      - `수계 직접 투입`: NaOH 용액이 수계에 직접 들어가 희석과 pH 상승을 함께 만든다고 해석합니다.
      - `사포니피케이션`: NaOH가 fresh organic의 pre-saponification 조건을 만든다고 해석합니다. 현장 raw-feed replay는 이 해석이 더 적합할 수 있습니다.
    - **고정 NaOH + 사포니피케이션 해석**:
      - 이 경로는 입력 NaOH 조건을 raw-feed 기준 **equivalent cascade target pH**로 환산하는 field-calibrated fallback을 사용합니다.
      - 현장 Data1~6과 유사한 범위에서는 replay 용도로 유용하지만, 범위를 크게 벗어난 일반 예측에서는 보수적으로 해석하는 편이 안전합니다.
    - **NaOH 희석 self-consistent 추정**:
      - 목표 pH 모드에서 선택적으로 켤 수 있는 고급 옵션입니다.
      - 활성화하면 사용자가 입력한 NaOH 농도(M)를 기준으로, 목표 pH 달성에 필요한 NaOH 유량과 그에 따른 수계 희석까지 함께 계산합니다.
      - 현장 프리셋(Data1~6)은 과거 검증과의 일관성을 위해 기본 OFF 해석이 더 안전합니다.
6.  **📐 모델 파라미터 편집**
    - `pH50`, `k`, `E_max`, `n_H`, `alpha`, `beta`, `gamma`를 직접 수정할 수 있습니다.
    - 이 영역은 결과를 크게 바꾸므로, 먼저 프로필을 선택한 뒤 필요한 경우에만 수정하는 편이 안전합니다.
    - `🔄 선택 프로필로 초기화` 버튼을 누르면 현재 선택한 프로필 기준으로 되돌아갑니다.

### B. 우측 대시보드 (결과 탭 활용법)

- **📊 시뮬레이션 결과**:
  - 전반적인 각 금속의 추출률(%)과 남은 후액(Raffinate)의 농도를 요약해서 봅니다.
  - **가장 먼저 볼 항목**은 `수렴 상태`, `검증 커버리지`, `최대 로딩률`, `후액 최종 pH`입니다.
  - 최대 로딩률이 높고, 검증 커버리지가 낮으면 수치는 맞더라도 과신하면 안 됩니다.
- **📎 검증 커버리지**:
  - 현재 입력이 Data1~6 현장 검증 범위와 얼마나 겹치는지 알려주는 안내 영역입니다.
  - `현장 검증 범위 안`, `부분 검증 범위`, `검증 범위 밖`으로 표시됩니다.
  - 이 표시는 정답/오답 판정이 아니라, **현재 결과를 얼마나 보수적으로 해석해야 하는지 알려주는 안전 가이드**입니다.
- **📉 McCabe-Thiele**: 가장 중요한 공학 탭입니다. '조작선(붉은 점선)'과 '평형선(파란 실선)' 사이를 오가는 '초록색 계단'이 시뮬레이터가 계산한 실제 역류 추출(Counter-current) 단수 궤적입니다.
- **🔬 추출제 비교**:
  - 동일한 수계 원수를 Cyanex 272와 D2EHPA에 각각 투입했을 때의 효율 차이를 나란히 비교합니다.
  - 비교 탭 상단에는 현재 공통 비교 조건(`C_ext`, 단수, 온도, 황산염)이 표시됩니다. 비교 결과는 이 조건을 기준으로 해석해야 합니다.
- **📋 상세 데이터**: 단별로(stage 1, 2, 3...) 추출액과 후액의 농도가 어떻게 변화해가는지 엑셀 표 형태로 정밀 분석합니다.
- **📜 변경 이력**: 현재 앱 버전과 버전별 변경사항을 확인할 수 있습니다.
- **📘 사용자 매뉴얼**: 현재 사용 중인 버전 기준 사용 지침을 확인할 수 있습니다.

## 3. 결과 해석 순서

처음 사용할 때는 아래 순서로 해석하는 것을 권장합니다.

1. **프리셋/입력값이 의도한 조건인지 확인**
2. **수렴 상태가 `수렴`인지 확인**
3. **검증 커버리지가 `현장 검증 범위 안`인지, `부분 검증 범위`인지 확인**
4. **최대 로딩률이 85%를 넘는지 확인**
5. **금속별 추출률과 후액 농도를 확인**
6. 필요하면 **McCabe-Thiele**, **상세 데이터**, **추출제 비교** 탭으로 내려가 원인을 해석

## 4. 고급 분석 팁 및 트러블슈팅

### 금속이 원하는 대로 추출되지 않을 때 (Extraction Failure)

1. **추출제 양 부족 (Maximum Loading Reached)**:
   - 메인 결과 창 우측 상단의 **최대 로딩률**을 확인하세요. 이 수치가 85~100%에 달했다면, 좁은 공간에 금속이 너무 많아 우선순위가 뒤쳐진 금속(예: Ni, Mg)들이 쫓겨났다는 뜻입니다. (시뮬레이터 내부 'Phase 3 다핵 착물 방어 알고리즘' 작동 중)
   - **해결책**: 추출제 농도를 두 배 높이거나, O/A 비율을 크게 올려 추출제 풀의 여유 공간을 확보해 보세요.
2. **목표 pH 부족 또는 종분화 저항성 (Speciation Buffering)**:
   - 금속이 수산화 착물로 전환되며 pH가 뻗어나가지 못하는 현상이 내부 발생했을 수 있습니다 (Phase 3 종분화 효과). 더 높은 목표 pH를 입력해보십시오.
   - 목표 pH 모드에서 **NaOH 희석 self-consistent 추정**을 켠 경우, 가정한 NaOH 농도가 낮을수록 필요 유량이 커지고 후액 희석도 커집니다. 현장 NaOH 농도 가정이 맞는지 함께 확인해야 합니다.
3. **검증 범위 밖 조건**:
   - `검증 커버리지`가 낮게 뜬다면, 수치 자체가 틀렸다는 뜻은 아니지만 현장 검증 데이터가 거의 없는 조건일 수 있습니다.
   - **해결책**: 먼저 문헌 기본값과 현장 보정 프로필을 둘 다 돌려 보고, 두 결과 차이가 큰지 확인하세요. 차이가 크면 그 조건은 추가 실험 검증이 필요합니다.
4. **미수렴 조건**:
   - 결과 화면에서 `수렴 상태`가 `미수렴`이면, 숫자는 표시되더라도 직접 의사결정에 사용하지 않는 편이 안전합니다.
   - **해결책**: 단수, pH, NaOH 전략, 추출제 농도, O/A 비를 완화해서 다시 시도하세요.

### 현장 실험과 결과가 불일치할 때 (튜닝의 기초)

- 사용하시는 용매(희석제) 특성상 현장 피팅 데이터보다 더 낮은 pH에서 추출이 시작될 수도 있습니다.
- **"📝 데이터 피팅"** 탭을 통해 현장 비커 샘플(Isotherm)에서 얻어진 실험 결과(pH vs 추출률) CSV 파일을 직접 업로드해 보세요. 시뮬레이터가 해당 사업장 고유의 추출 파라미터($pH_{50}$) 곡선을 다시 그려내어 드리며, 클릭 한 번에 앱에 즉시 적용 가능합니다.
- 다만, **한 현장 데이터에만 맞춘 피팅은 다른 현장 일반화 성능을 떨어뜨릴 수 있습니다.**
- 가능하면 피팅 데이터와 검증 데이터를 나누어 보고, `현장 보정`과 `문헌 기본값` 결과를 함께 비교하는 습관을 권장합니다.

## 5. 사용 시 주의사항

- 이 앱은 현재 **현장 범위 내 공정 해석용 시뮬레이터**에 가깝습니다.
- `현장 보정 (Isd 108)` 프로필은 특정 현장 조건 재현에 유리하지만, 다른 사업장까지 자동으로 일반화된다고 가정하면 안 됩니다.
- `문헌 기본값` 프로필은 현장 적합도는 낮을 수 있지만, 특정 현장 보정에 덜 묶인 기준선으로 유용합니다.
- 검증 범위를 벗어난 고이온강도, 고로딩, 고온, 다른 희석제 조건은 **정량 예측보다 경향 확인 용도**로 쓰는 편이 안전합니다.
- 목표 pH 모드에서 **NaOH 희석 self-consistent 추정**을 켰다면, 결과는 입력한 NaOH 농도 가정에 민감합니다. 실제 설비 NaOH 농도가 불확실할 때는 OFF 결과와 함께 비교하는 편이 안전합니다.
