# 📘 Li/Ni/Co/Mn Mixer-Settler SX 시뮬레이터 사용자 매뉴얼

처음 시뮬레이터를 접하시는 연구원님들을 환영합니다! 본 매뉴얼은 시뮬레이터의 목적, 각 화면 요소의 기능, 그리고 고품질의 예측 결과를 얻기 위한 파라미터 튜닝 노하우를 담고 있습니다.

---

## 1. 개요 및 활용 목적

본 시뮬레이터는 배터리 재활용 등 습식제련(Hydrometallurgy) 공정에서 여러 금속(Li, Ni, Co, Mn, Ca, Mg, Zn)이 혼합된 수용액으로부터 특정 금속을 선택적으로 분리해내는 **용매 추출(Solvent Extraction, SX)** 공정을 예측합니다.

- **활용 목적**:
  - **조업 조건 도출**: O/A 비, 단수(Stages), 초기 pH, NaOH 투입량 등 최적의 조업 조건을 현장 실험 전에 사전 시뮬레이션합니다.
  - **스케일업 및 문제 해결**: 현장에서 예상치 못하게 추출률이 떨어지는 원인(예: 추출제 부족, 종분화 완충 등)을 모델을 통해 진단합니다.
  - **추출제 성능 비교**: Cyanex 272와 D2EHPA 중 어떤 추출제가 현재 원수 비율에 적합한지 빠르게 판별합니다.

---

## 2. 화면 구성 및 핵심 조작법

화면은 크게 **좌측 사이드바(입력부)**와 **우측 메인 대시보드(결과 출력부)**로 나뉩니다.

### A. 좌측 사이드바 (입력 및 설정)

시뮬레이션 구동에 필요한 모든 환경 변수를 조작합니다.

1.  **🧪 현장 실험 Data 프리셋**
    - **가장 먼저 테스트해 볼 항목입니다.** 연구개발 그룹에서 사전에 검증을 마친 현장 데이터(Data1~6) 환경을 원클릭으로 불러올 수 있습니다. 프리셋을 변경하면 아래의 모든 농도와 단수 설정이 그에 맞게 자동 세팅됩니다.
2.  **🚰 Feed (수계) 조건**
    - 반입되는 원수의 각 금속의 초기 농도(g/L)를 입력합니다.
    - 황산염($\text{SO}_4^{2-}$) 농도는 입력된 금속들을 기준으로 100% 황산염 염(Salt) 형태로 결합되어 있다고 가정하여 시스템이 **자동 연산**하므로 직접 입력하실 필요가 없습니다. (수계 버퍼 용량 계산에 사용됨)
3.  **🛢️ Organic (유기계) 조건**
    - **추출제 선택**: Cyanex 272(코발트 우선 추출)와 D2EHPA(불순물 우선 추출) 중 실험 목적에 맞는 추출제를 선택합니다.
    - **O/A 비율 (Phase Ratio)**: 유기상(Organic)과 수계상(Aqueous)의 부피 투입 비율입니다. 1.0은 동량 투입을 의미하며, 농축이 필요할 땐 0.5 근처로 낮추기도 합니다.
4.  **⚙️ 반응 설비 및 타겟 조건**
    - **Mixer-Settler 단수 (추출단)**: 실제 공정에 설치될 반응조의 개수입니다. 단수가 늘어날수록 추출 한계치에 수렴합니다.
    - **운전 온도**: 온도는 반응 속도와 화학양론에 미미하지만 영향을 줍니다.
    - **pH 제어 방식**: "고정 NaOH 투입 모드"와 "목표 pH (자동 역산)" 모델 중 하나를 택합니다. **초심자는 "목표 pH 자동 역산" 모드를 적극 권장**합니다.

### B. 우측 대시보드 (결과 탭 활용법)

- **📊 시뮬레이션 결과**: 전반적인 각 금속의 추출률(%)과 남은 후액(Raffinate)의 농도를 요약해서 봅니다. 최대 로딩률 포화도 경고를 체크하세요.
- **📉 McCabe-Thiele**: 가장 중요한 공학 탭입니다. '조작선(붉은 점선)'과 '평형선(파란 실선)' 사이를 오가는 '초록색 계단'이 시뮬레이터가 계산한 실제 역류 추출(Counter-current) 단수 궤적입니다.
- **🔬 추출제 비교**: 동일한 수계 원수를 Cyanex 272와 D2EHPA에 각각 투입했을 때의 효율 차이를 나란히 비교합니다.
- **📋 상세 데이터**: 단별로(stage 1, 2, 3...) 추출액과 후액의 농도가 어떻게 변화해가는지 엑셀 표 형태로 정밀 분석합니다.

## 3. 고급 분석 팁 및 트러블슈팅

### 금속이 원하는 대로 추출되지 않을 때 (Extraction Failure)

1. **추출제 양 부족 (Maximum Loading Reached)**:
   - 메인 결과 창 우측 상단의 **최대 로딩률**을 확인하세요. 이 수치가 85~100%에 달했다면, 좁은 공간에 금속이 너무 많아 우선순위가 뒤쳐진 금속(예: Ni, Mg)들이 쫓겨났다는 뜻입니다. (시뮬레이터 내부 'Phase 3 다핵 착물 방어 알고리즘' 작동 중)
   - **해결책**: 추출제 농도를 두 배 높이거나, O/A 비율을 크게 올려 추출제 풀의 여유 공간을 확보해 보세요.
2. **목표 pH 부족 또는 종분화 저항성 (Speciation Buffering)**:
   - 금속이 수산화 착물로 전환되며 pH가 뻗어나가지 못하는 현상이 내부 발생했을 수 있습니다 (Phase 3 종분화 효과). 더 높은 목표 pH를 입력해보십시오.

### 현장 실험과 결과가 불일치할 때 (튜닝의 기초)

- 사용하시는 용매(희석제) 특성상 현장 피팅 데이터보다 더 낮은 pH에서 추출이 시작될 수도 있습니다.
- **"📝 데이터 피팅"** 탭을 통해 현장 비커 샘플(Isotherm)에서 얻어진 실험 결과(pH vs 추출률) CSV 파일을 직접 업로드해 보세요. 시뮬레이터가 해당 사업장 고유의 추출 파라미터($pH_{50}$) 곡선을 다시 그려내어 드리며, 클릭 한 번에 앱에 즉시 적용 가능합니다.
