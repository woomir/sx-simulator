# v2.1.1 6차 리팩토링 기록

## 목적

6차 리팩토링의 목적은 `sx_dashboard.py`에 남아 있던 큰 탭 블록 중 수식 탭과 데이터 피팅 탭을 별도 렌더러 함수로 분리하는 것이다.

이번 변경은 다음 문제를 줄이기 위한 작업이다.

- 대시보드 파일에 수식 설명용 긴 UI 블록이 남아 있어 읽기 흐름이 끊기던 문제
- 피팅 탭의 CSV 업로드, 피팅 계산, 파라미터 적용 UI가 메인 파일에 직접 섞여 있던 문제
- 탭 단위 수정 시 충돌 범위가 커지고 검토 난이도가 올라가던 문제

## 변경 파일

- `sx_dashboard.py`
  - 수식 탭과 피팅 탭 본문을 직접 렌더링하지 않고 `dashboard_tabs.py`의 함수 호출만 하도록 정리
  - 수식/피팅 탭에서만 쓰던 import를 메인 파일에서 제거
- `sx_simulator/dashboard_tabs.py`
  - `render_formula_tab()`
  - `render_fitting_tab()`
  - 위 두 함수 추가
  - 수식 탭에서 쓰는 isotherm 계산 helper, 피팅 탭에서 쓰는 fitting helper import 정리
- `PROJECT_STRUCTURE.md`
  - 현재 구조와 리팩토링 문서 목록 반영

## 구조 변화

변경 전:

- `sx_dashboard.py`가 결과 탭, 수식 탭, 피팅 탭을 직접 렌더링

변경 후:

- `sx_dashboard.py`는 입력 수집과 탭 연결에 집중
- `sx_simulator/dashboard_tabs.py`가 결과/수식/피팅을 포함한 주요 동적 탭 렌더링을 담당

## 동작 보존 원칙

이번 리팩토링은 구조 정리만 수행했다.

- 시뮬레이션 엔진 계산 로직은 변경하지 않음
- 파라미터 값과 보정 규칙은 변경하지 않음
- 피팅 탭의 업로드 형식, 피팅 절차, 세션 상태 반영 방식은 유지
- 수식 탭의 수식 표시 내용과 현재 설정값 반영 방식은 유지

## 회귀 확인

아래 항목으로 동작 보존을 다시 확인했다.

- `python3 -m py_compile`
  - `sx_dashboard.py`
  - `sx_simulator/dashboard_service.py`
  - `sx_simulator/dashboard_tabs.py`
  - `sx_simulator/extraction_isotherm.py`
  - `sx_simulator/multistage_sx.py`
  - `sx_simulator/single_stage.py`
- `validation_test.py` 통과
- `test_verification.py` 실행 결과의 주요 오차 패턴 유지
  - `Data1 Ni`: 약 `-6.397 g/L`
  - `Data2 Ni`: 약 `-2.996 g/L`
  - `Data4 Li`: 약 `+2.160 g/L`
  - `Data6 Co`: 약 `-0.845 g/L`
- Streamlit 앱 부팅 확인

즉 6차 리팩토링은 UI 코드 배치만 바꾸고, 웹앱 기능과 시뮬레이션 결과는 유지하는 방향으로 마무리했다.
