### #1. [Critical] 고정 NaOH 모드 로딩 감쇠 내부 반복 부재

**현상**
`_solve_with_fixed_NaOH()` (single_stage.py L270-273)에서 bisection 반복 시 이전 pH 반복의 `C_org_out`으로 로딩률을 계산합니다.
`_solve_at_target_pH()`에서는 로딩 감쇠 반복(L162-200, 최대 30회)이 있지만, `_solve_with_fixed_NaOH()`에서는 단일 패스만 수행합니다.
고로딩 조건에서 고정 NaOH 모드 사용 시 부정확한 결과가 나올 수 있습니다.

또한 검증에서 고정 NaOH 모드가 500회 반복에도 미수렴(converged=False)한 것이 확인되었습니다.
결과 자체는 안정적이었으나, 수렴 판정 개선이 필요합니다.

**개선 방안**
`_solve_with_fixed_NaOH()`의 비경쟁 모드(else 분기) 내부에 로딩 감쇠 내부 반복을 추가합니다.

```python
# single_stage.py L270~ (비경쟁 모드 분기 내부)
# 변경 전:
loading_frac = calc_loading_fraction(
    C_org_out if C_org_out else C_org_in, extractant, C_ext, metals)
damping = loading_damping_factor(loading_frac)

# 변경 후:
# 로딩 감쇠 내부 반복 (target_pH 모드와 동일한 패턴)
damping = 1.0
for _load_iter in range(15):
    C_aq_out_tmp = {}
    C_org_out_tmp = {}
    for metal in metals:
        D_raw = distribution_coefficient(
            pH_current, metal, extractant, C_ext, temperature=temperature)
        D = D_raw * damping
        MW = MOLAR_MASS[metal]
        C_aq_mol_in = C_aq_in.get(metal, 0.0) / MW
        C_org_mol_in = C_org_in.get(metal, 0.0) / MW
        total_metal_flow = C_aq_mol_in * Q_aq + C_org_mol_in * Q_org
        denominator = Q_aq_eff + D * Q_org
        C_aq_mol_out = total_metal_flow / denominator if denominator > 0 else 0.0
        C_org_mol_out = D * C_aq_mol_out
        C_aq_out_tmp[metal] = C_aq_mol_out * MW
        C_org_out_tmp[metal] = C_org_mol_out * MW

    new_loading = calc_loading_fraction(C_org_out_tmp, extractant, C_ext, metals)
    new_damping = loading_damping_factor(new_loading)
    if abs(new_damping - damping) < 1e-4:
        damping = new_damping
        break
    damping = 0.5 * new_damping + 0.5 * damping

# 최종 damping으로 결과 확정
for metal in metals:
    # ... (기존 L275-298과 동일한 로직을 damping 확정 후 1회 수행)
```

**다단 수렴 개선 (multistage_sx.py)**
고정 NaOH 모드에서 relaxation factor의 적응형 전략을 개선합니다.

```python
# multistage_sx.py L126
# 변경 전:
alpha = 0.3 if iteration < 50 else 0.7

# 변경 후: 3단계 적응형
if iteration < 30:
    alpha = 0.2    # 초기: 매우 보수적
elif iteration < 100:
    alpha = 0.5    # 중기: 점진적 증가
else:
    alpha = 0.8    # 후기: 적극적
```

**영향 범위**: `single_stage.py` (`_solve_with_fixed_NaOH`), `multistage_sx.py` (relaxation)

```python
                              C_ext: float, ao_ratio: float = 1.0,

# 변경 후:
def distribution_coefficient(pH: float, metal: str, extractant: str,
```

### #3. [Major] 목표 pH 모드 NaOH 희석(Q_aq_eff) 미반영

**현상**
`_solve_at_target_pH()` (single_stage.py L140)에서 `Q_aq_eff=Q_aq`으로 경쟁 추출에 전달합니다.
목표 pH 모드는 NaOH 필요량을 **역산**하는 구조이므로, NaOH 유량을 사전에 알 수 없어 희석을 반영할 수 없는 구조적 한계가 있습니다.

**개선 방안**
NaOH 역산 결과로부터 희석 보정을 위한 2차 반복을 추가합니다.

```python
# single_stage.py _solve_at_target_pH 함수 말미
# NaOH 역산 후, 추정된 NaOH 유량으로 Q_aq_eff를 보정하고 재계산

# 1차 계산: Q_aq_eff = Q_aq (현재 로직)
NaOH_consumed = ...  # 역산 결과 (mol/hr)

# NaOH 농도를 가정하여 유량 추정 (예: 10M NaOH)
C_NaOH_assumed = 10.0  # M (농축 NaOH 가정)
Q_NaOH_estimated = NaOH_consumed / C_NaOH_assumed
Q_aq_eff_corrected = Q_aq + Q_NaOH_estimated

# 희석 보정이 유의미한 경우(Q_NaOH > Q_aq의 5%)에만 2차 보정
if Q_NaOH_estimated > Q_aq * 0.05:
    # 2차 계산: 보정된 Q_aq_eff로 재계산
    # ... (동일 로직을 Q_aq_eff_corrected로 재실행)
```

**대안 (더 간단)**: 이 한계를 docstring과 대시보드 UI에 명시하고, 정밀도가 필요한 경우 고정 NaOH 모드를 사용하도록 안내. 현재 물질수지 오차가 0.001% 이하이므로, 실용적으로 문제가 되지 않을 수 있습니다.

**권장**: 대안(문서화)을 우선 적용하고, 향후 필요 시 2차 반복을 추가합니다.

**영향 범위**: `single_stage.py` (`_solve_at_target_pH`)

### #4. [Major] pH > 7 OH⁻ 보정 Q_aq/Q_aq_eff 비대칭

**현상**
- `_solve_at_target_pH` L213-214: `Q_aq` 사용
- `_solve_with_fixed_NaOH` L310-311: `Q_aq_eff` 사용

목표 pH 모드에서는 NaOH 유량을 모르므로 Q_aq를 사용하는 것이 불가피하지만, 이를 #3의 보정과 연계하여 개선할 수 있습니다.

**개선 방안**
#3의 2차 보정이 적용되지 않는 경우(대안 선택 시), 최소한 docstring에 이 비대칭을 명시합니다.

#3의 2차 보정이 적용되는 경우, 2차 계산에서 Q_aq_eff_corrected를 OH⁻ 보정에도 사용합니다:

```python
# 변경 전:
if target_pH > 7.0:
    total_H_out_target -= Q_aq * (10.0 ** -(14.0 - target_pH))

# 변경 후 (2차 보정 적용 시):
if target_pH > 7.0:
    total_H_out_target -= Q_aq_eff * (10.0 ** -(14.0 - target_pH))
```

**영향 범위**: `single_stage.py` L213-214

### #5. [Major] n_eff 계산에서 `C_ext/2.0` 하드코딩

**현상**
`compute_competitive_extractions()` (extraction_isotherm.py L348)에서 최대 이론 적재량을 `C_ext / 2.0`으로 계산합니다.
이는 모든 금속의 `n_ext = 2`를 가정하지만, Li는 `n_ext = 1`입니다.

**개선 방안**
실제 금속 구성에 따른 가중 평균 n_ext를 사용합니다.

```python
# extraction_isotherm.py L345-348
# 변경 전:
max_theoretical_M_mol = C_ext / 2.0

# 변경 후: 현재 유기상 금속 구성 기반 가중 평균 n_ext 사용
if total_M_org_in_mol > 0:
    weighted_n_ext = 0.0
    for m in metals:
        C_org_m = C_org_in.get(m, 0.0)
        if C_org_m > 0:
            mol_m = C_org_m / MOLAR_MASS[m]
            n_ext_m = EXTRACTANT_PARAMS[extractant][m].get("n_ext", 2)
            weighted_n_ext += n_ext_m * (mol_m / total_M_org_in_mol)
    weighted_n_ext = max(weighted_n_ext, 1.0)  # 최소 1.0 보장
else:
    weighted_n_ext = 2.0  # 기본값 (유기상 비어있을 때)

max_theoretical_M_mol = C_ext / weighted_n_ext
```

**영향 범위**: `extraction_isotherm.py` L345-348. Data5(Ni 지배적, n_ext=2)에는 변화 없음. Li가 유기상의 주 금속인 시나리오에서만 차이 발생.

**검증 필요**: 변경 후 Data1~6 프리셋 전체 재검증 필수.

### #7. [Minor] 발산 감지 로직 추가

**현상**
`multistage_sx.py` L86-143의 역류 반복에서 relaxation factor로 안정화를 시도하지만, 발산 시 단순히 500회 반복 후 종료됩니다.

**개선 방안**
연속 반복에서 max_diff가 증가하는 패턴을 감지하여 relaxation을 자동 조정합니다.

```python
# multistage_sx.py, for iteration 루프 내부 (L136-143 부근)

prev_max_diff = getattr(prev_max_diff, 'val', float('inf'))  # 이전 반복의 max_diff

if max_diff < tolerance:
    converged = True
    break

# 발산 감지: 연속 3회 이상 max_diff 증가 시 alpha 감소
if not hasattr(_run_with_q_naoh, '_diverge_count'):
    _run_with_q_naoh._diverge_count = 0

if iteration > 0 and max_diff > prev_max_diff * 1.1:
    _run_with_q_naoh._diverge_count += 1
    if _run_with_q_naoh._diverge_count >= 3:
        alpha = max(0.1, alpha * 0.5)  # relaxation 강화
        _run_with_q_naoh._diverge_count = 0
else:
    _run_with_q_naoh._diverge_count = 0

prev_max_diff = max_diff
```

**대안 (더 간단)**:

```python

# 변경 전:

    alpha = 0.2
elif iteration < 100:
    alpha = 0.5

    alpha = 0.7
else:
    alpha = 0.9  # 후반부 적극적 수렴 시도
```

**권장**: 대안(간단한 alpha 전략 개선)을 우선 적용합니다.

### #13. [Minor] `EXTRACTANT_PARAMS` 모듈 레벨 직접 수정 안전화

**현상**
`sx_dashboard.py` L233-237에서 `EXTRACTANT_PARAMS` 모듈 전역 변수를 직접 수정합니다. Streamlit의 모듈 캐싱 특성상 세션 간 상태 오염이 발생할 수 있습니다.

**개선 방안**
모듈 전역을 수정하는 대신, 시뮬레이션 호출 시 로컬 복사본을 사용합니다.

```python
# sx_dashboard.py L233-237
# 변경 전: EXTRACTANT_PARAMS를 직접 수정
for ext_name in st.session_state.custom_params:
    for metal in st.session_state.custom_params[ext_name]:
        for k, v in st.session_state.custom_params[ext_name][metal].items():
            EXTRACTANT_PARAMS[ext_name][metal][k] = v

# 변경 후: 시뮬레이션 전에 임시로 적용하고 복원
import copy
_original_params = copy.deepcopy(EXTRACTANT_PARAMS)
try:
    for ext_name in st.session_state.custom_params:
        for metal in st.session_state.custom_params[ext_name]:
            for k, v in st.session_state.custom_params[ext_name][metal].items():
                EXTRACTANT_PARAMS[ext_name][metal][k] = v
    # ... 시뮬레이션 실행 ...
finally:
    # 원래 값 복원
    for ext_name in _original_params:
        for metal in _original_params[ext_name]:
            EXTRACTANT_PARAMS[ext_name][metal] = _original_params[ext_name][metal]
```

**대안 (더 간단)**: 현재 구조를 유지하되, "기본값으로 초기화" 버튼(L239-244)이 이미 있으므로 이 기능이 작동하는지 확인하고 문서화합니다. Streamlit의 `st.cache_resource`가 사용되지 않는 한 각 세션이 독립적이므로 실질적 문제가 제한적입니다.

**권장**: 대안(현재 구조 유지 + 확인)을 적용합니다.
