# v2.1.1 시뮬레이션 개선 누적 로그

이 문서는 시뮬레이션 엔진 관련 변경을 단계별로 누적 기록한다.

원칙:

- 한 번에 한 가지 축만 변경한다.
- 각 단계마다 의도와 범위를 명확히 남긴다.
- 계산 결과가 의도적으로 바뀌는 단계인지, 구조 정리만 하는 단계인지 구분한다.
- 각 단계의 검증 결과는 별도 문서 `v2.1.1_시뮬레이션_검증_누적로그.md`에 누적한다.

---

## 1단계. 파라미터 체계 분리

### 목적

현 시뮬레이터는 `field_calibrated`와 `literature_default` 프로필을 지원하지만, 코드 구조상 현장 보정값이 기본 딕셔너리에 섞여 있어 일반화 범위를 판단하기 어려웠다.

이번 단계의 목적은 아래 두 가지다.

- 문헌 기본값과 현장 보정값을 코드상에서 명확히 분리
- 런타임 기본 동작은 유지하여 시뮬레이션 결과를 바꾸지 않음

### 변경 파일

- `sx_simulator/config.py`

### 변경 내용

1. `LITERATURE_BASE_PARAMS`를 신설했다.
   - 문헌/기본 추정값을 보관하는 원본 파라미터 세트다.
2. `SITE_PARAMETER_OVERRIDES`를 신설했다.
   - 현장 보정값은 override로 별도 관리한다.
3. `_build_parameter_profile()` helper를 추가했다.
   - 기본 파라미터와 override를 조합해 프로필을 생성한다.
4. `FIELD_CALIBRATED_PARAMS`는 이제 `LITERATURE_BASE_PARAMS + SITE_PARAMETER_OVERRIDES`로 생성된다.
5. `EXTRACTANT_PARAMS`는 하위 호환용 기본 별칭으로 유지했다.
   - 엔진이 명시적 파라미터 세트를 받지 않을 때의 기본 동작은 기존과 동일하다.
6. `get_parameter_profile("literature_default")`는 이제 현장 보정값을 되돌리는 방식이 아니라, 문헌 기본값 복사본을 직접 반환한다.

### 현재 프로필 차이

이번 단계에서 코드상으로 명시된 현장 오버레이는 아래 두 항목이다.

- `Cyanex 272 / Ni / pH50`: `5.8 -> 6.3`
- `D2EHPA / Mg / pH50`: `4.5 -> 3.2`

### 기대 효과

- 앞으로 어떤 값이 범용 기본값이고 어떤 값이 현장 보정값인지 즉시 파악 가능
- 다음 단계에서 `site-specific` 보정 범위를 체계적으로 확장 가능
- 일반화 성능 검토 시 프로필 차이의 출처를 코드에서 직접 추적 가능

### 비의도적 변경 방지 기준

이번 단계는 모델 구조 개선이 아니라 파라미터 계층 구조 정리다.

- 시뮬레이션 기본 결과는 유지되어야 한다.
- 현장 검증 오차 패턴은 이전과 동일해야 한다.
- `validation_test.py`의 구조 검증은 그대로 통과해야 한다.

### 다음 단계 후보

2단계는 아래 둘 중 하나가 유력하다.

- `K_MSO4`를 실제 계산에 연결하는 황산염 착물/자유 금속 보정
- `목표 pH` 모드의 NaOH 희석 self-consistent 보정

정확도 개선 효과는 황산염 착물 반영 쪽이 더 클 가능성이 높다.

## 2단계. 황산염 착물/자유 금속 speciation 연결

### 목적

`config.py`에는 `K_MSO4`가 정의되어 있었지만, 실제 계산에서는 거의 사용되지 않았다.

이번 단계의 목적은 아래와 같다.

- 황산(HSO4-) 평형과 별개로 금속 황산염 착물(`MSO4^0`) 효과를 코드에 연결
- 자유 금속 분율이 낮아지면 hydrolysis(`MOH+`)도 줄어들도록 반영
- 아직 추출 D 자체는 건드리지 않고, 먼저 speciation/pH submodel을 정합화

### 변경 파일

- `sx_simulator/extraction_isotherm.py`
- `sx_simulator/single_stage.py`

### 변경 내용

1. `get_free_sulfate_concentration()` 추가
   - 총 황산염 농도와 pH로부터 자유 `SO4^2-` 농도를 계산한다.
2. `get_aqueous_speciation_state()` 추가
   - 단순 1:1 가정에서 금속별 `free / hydroxo / sulfate` 종분율을 계산한다.
3. `calc_aq_protons()` 개선
   - 기존에는 `K_MOH`만 사용해 hydrolysis를 계산했다.
   - 이제 `K_MSO4`까지 반영한 종분율에서 `hydroxo_fraction`을 직접 사용한다.
   - 즉, 황산염 착물이 강한 조건에서는 자유 금속이 줄고, 그에 따라 `MOH+` 생성도 줄어든다.

### 이번 단계에서 일부러 하지 않은 것

이번 단계에서는 아래를 의도적으로 보류했다.

- `distribution_coefficient()` 직접 보정
- `compute_competitive_extractions()`의 D 계산에 sulfate factor 삽입

이유는 현재 추출 모델이 경험식 `pH50/k/E_max` 기반이라, 황산염 보정을 바로 D에 곱하면 기존 현장 보정값과 중복될 가능성이 크기 때문이다.

즉 이번 단계는 먼저 `speciation submodel`을 물리적으로 맞추는 작업이다.

### 기대 효과

- 고황산염 조건에서 자유 금속 분율과 hydrolysis 계산의 물리적 일관성 향상
- 이후 단계에서 `free metal` 기반 추출 보정을 넣을 수 있는 기반 확보
- `K_MSO4`가 설정만 존재하는 죽은 상수 상태 해소

### 한계

이번 단계만으로는 현장 후액 농도 오차가 바로 개선되지 않을 수 있다.

특히 `test_verification.py`는 대부분 `목표 pH` 모드에서 돌아가므로, 이번 단계의 영향은 주로 `NaOH/pH submodel`과 고정 NaOH 모드 쪽에서 먼저 나타난다.

### 다음 단계 후보

3단계는 아래 두 방향 중 하나다.

- `free metal fraction`을 추출 D에 상대 보정 형태로 연결
- `목표 pH` 모드에서 NaOH 희석 self-consistent 반복 도입

정량 정확도 개선만 보면 첫 번째가 더 직접적이고, 구조적 일관성만 보면 두 번째가 더 중요하다.

## 3단계. D2EHPA-Co 고황산염 상대 보정

### 목적

2단계에서 `K_MSO4`를 speciation 계산에 연결했지만, 현장 후액 농도는 거의 변하지 않았다.

그 이유는 현재 추출 모델이 경험식 `pH50/k/E_max` 기반이라, sulfate 효과를 모든 금속에 일괄 곱하면 기존 현장 보정값과 충돌하기 때문이다.

이번 단계의 목적은 아래와 같다.

- 전 금속 일괄 보정이 아니라, 실제 검증에서 문제로 드러난 조합만 선택 보정
- D2EHPA 고황산염 조건에서 Co 과추출을 줄여 현장 데이터 정합도 개선
- Co 로딩 완화에 따른 Ni 거동 개선이 있는지 함께 확인

### 변경 파일

- `sx_simulator/config.py`
- `sx_simulator/extraction_isotherm.py`
- `sx_simulator/single_stage.py`

### 변경 내용

1. `SULFATE_D_CORRECTION_RULES` 추가
   - 현재는 `("D2EHPA", "Co")` 조합만 활성화
2. `get_sulfate_d_correction_factor()` 추가
   - 2단계에서 만든 상대 자유 금속 계수를 기반으로 D 보정 계수를 생성
   - 규칙이 없는 금속/추출제 조합은 보정하지 않음
3. D 적용 위치
   - 일반 분배 경로 `_partition_stage_with_damping()`
   - 경쟁 추출 경로 `compute_competitive_extractions()`
4. 현재 적용식
   - `raw_factor = relative_free_metal_factor`
   - `correction = clamp(raw_factor ** 4, 0.2, 1.0)`

### 설계 이유

- 초기 시도처럼 모든 금속에 sulfate 상대 보정을 적용하면 Ni 오차가 크게 악화됐다.
- 반면 D2EHPA-Co만 선택적으로 억제하면:
  - Co 과추출이 줄고
  - 유기상 로딩 경쟁이 완화되어
  - 일부 고황산염 조건에서 Ni도 같이 개선됐다.

즉 이번 단계는 `전체 물리모델 일반화`보다 `현장 오차 패턴을 망치지 않는 범위의 선택 보정`에 가깝다.

### 현재 보정 거동

예시:

- `D2EHPA / Co / sulfate 0.7 M` → 보정 계수 `1.0`
- `D2EHPA / Co / sulfate 1.0 M` → 보정 계수 `1.0`
- `D2EHPA / Co / sulfate 1.5 M` → 보정 계수 약 `0.316`
- `Cyanex 272 / Co / sulfate 1.5 M` → 보정 없음 (`1.0`)
- `D2EHPA / Ni / sulfate 1.5 M` → 보정 없음 (`1.0`)

### 기대 효과

- Data4~6 같은 고황산염 D2EHPA 조건에서 Co 후액 예측 개선
- 과도한 Co 우선 추출 억제로 Ni 후액 예측도 일부 안정화
- 2단계의 speciation 계산을 실제 추출 계산으로 연결하는 첫 번째 제한적 적용 사례 확보

### 한계

- 아직 일반화된 sulfate 추출 보정 모델은 아니다.
- D2EHPA-Co 외 조합에는 동일한 방식이 유효한지 검증되지 않았다.
- 따라서 이 단계는 `선택적 보정`이며, 이후에는 metal/extractant별 독립 검증이 더 필요하다.

### 다음 단계 후보

4단계는 아래 두 방향 중 하나다.

- `target pH` 모드의 희석/알칼리 계산 구조를 정리
- 검증 스크립트를 허용오차 기반 자동 회귀 테스트로 강화

지금 시점에서는 엔진 정합도 개선과 함께 검증 자동화를 병행하는 편이 안전하다.

## 4단계. Cyanex 272-Ni 현장 재보정

### 목적

3단계까지 진행한 뒤에도 가장 큰 잔여 오차는 Cyanex 272 조건의 Ni 과추출이었다.

- `Data1 Ni`: `-6.397 g/L`
- `Data2 Ni`: `-2.996 g/L`

반면 `Data3`는 이미 충분히 맞고 있었다.

즉 문제는 `Ni 전체 모델이 틀렸다`기보다, 현장 프로필의 `Cyanex 272 / Ni` 전이 위치와 전이 폭이 저 pH 구간에서 아직 너무 공격적이라는 쪽에 가까웠다.

이번 단계의 목적은 아래와 같다.

- 별도 임시 보정식을 추가하지 않고
- 현장 오버레이 파라미터 자체를 다시 맞춰
- `Data1/2`를 개선하면서 `Data3`를 유지

### 변경 파일

- `sx_simulator/config.py`

### 변경 내용

`SITE_PARAMETER_OVERRIDES`의 `Cyanex 272 / Ni` 항목을 아래처럼 조정했다.

- `pH50`: `6.3 -> 6.55`
- `k`: `2.5 -> 2.0`

### 설계 이유

사전 grid 확인 결과, 저 pH Ni 과추출 문제는 별도 선택 보정보다 `pH50 + k` 재보정으로 더 깔끔하게 줄어들었다.

핵심 관찰:

- `pH50`만 올리면 `Data1`은 좋아지지만 `Data2`가 빠르게 과보정됐다.
- `k`를 함께 낮추면 `6.1`과 `5.89` 사이 전이 폭이 완만해져 `Data1/2`를 동시에 더 잘 맞출 수 있었다.
- `Data3`는 `pH 7.0`이라 여전히 충분히 높은 추출 영역에 있어 영향이 거의 없었다.

즉 이번 단계는 `새 알고리즘 추가`가 아니라, `현장 프로필의 Ni 전이 위치/기울기 재보정`이다.

### 기대 효과

- Cyanex 272 저 pH 조건에서 Ni 과추출 완화
- 고 pH Cyanex 272 조건(`Data3`) 유지
- 선택 보정 규칙 추가 없이 현장 프로필의 일관성 향상

### 한계

- 이번 단계는 여전히 `field_calibrated` 프로필 개선이다.
- 문헌 기본값(`literature_default`)의 일반화를 직접 개선하는 단계는 아니다.
- 따라서 다른 희석제나 다른 Cyanex 272 계 현장에 바로 일반화된다고 보기는 어렵다.

### 다음 단계 후보

5단계는 아래 두 방향 중 하나가 적절하다.

- `target pH` 모드의 NaOH 희석 self-consistent 계산 도입
- `test_verification.py`를 허용오차 기반 회귀 테스트로 강화
